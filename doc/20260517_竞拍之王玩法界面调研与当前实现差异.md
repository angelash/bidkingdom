# 20260517 竞拍之王玩法界面调研与当前实现差异

> 调研时间：2026-05-17  
> 对象：Steam App 4128580《BidKing / 竞拍之王》与当前 `bitkingdom` 内部试验 Demo。

## 1. 结论摘要

当前 Demo 已经覆盖了“4人竞拍、货柜估值、公共/私人线索、角色技能、开箱结算、图鉴、复盘后台”的骨架，但与《竞拍之王》的核心玩法相比，最关键差异不是市场、商城或仓型包装，而是**竞品围绕同一个仓库进行 5 轮渐进揭示，玩家通过明拍/暗拍的多轮出价反馈和角色技能推断最终合理成交价**。差异主要集中在四个方向：

1. 《竞拍之王》的核心是一仓 5 轮，当前 Demo 是 5 轮 5 个货柜。
2. 《竞拍之王》的信息揭示以“仓库格子/槽位 + 稀有度边框 + 轮廓 + 技能揭示”为主，当前 Demo 以“货柜卡片 + 文本线索 + 安全价 + 出价记录”为主。
3. 《竞拍之王》核心竞拍应先拆成明拍和暗拍两类；当前 Demo 并列了明拍、暗标、次高价、押金、闪拍，规则多但没有先把单仓五轮推价打透。
4. 《竞拍之王》的局外系统确实很重，但这些是周边和包装，不应进入当前核心复刻优先级。

建议后续不要先做市场、交易、商城、活动仓和复杂局前装备，而应先 100% 复刻“单仓五轮渐进揭示 + 明拍/暗拍 + 角色技能信息差 + 每轮出价反馈 + 最终开箱复盘”。仓型、题材和美术风格按包装层处理。

## 3. 《竞拍之王》玩法画像

### 3.1 核心循环

从视频帧和玩法结构推断，当前应优先复刻的核心循环是：

```text
选择角色
→ 进入同一个仓库
→ 第1轮获得初始仓库信息并出价
→ 第2-5轮逐步揭示更多格子/稀有度/轮廓/类别/估值信息
→ 每轮结合明拍/暗拍反馈修正心理价
→ 第5轮最高有效报价者成交
→ 开箱验证真实价值、成交价和盈亏
→ 复盘每轮信息、技能、出价和误判点
```

### 3.2 局内竞拍

公开资料能看到以下特征：

- 支持 4 人对局，强调读对手、虚张声势、低价拿高价值物。
- 视频帧显示 `Round 2`、`Auction Rounds 5`，说明至少有 5 回合结构。
- 竞拍前会展示仓库/货柜线索，例如随机若干藏品平均值、随机藏品稀有度、未知藏品轮廓等。
- 局内左侧显示 4 名玩家/收藏家席位、头像、编号、可能的出价/轮次状态。
- 右侧是 `Loot` 区，采用格子化宝物槽位，稀有度用白/绿/蓝/紫/金/红等颜色边框区分。
- 有 `Collection Codex` 入口，说明开箱结果与长期图鉴绑定。
- 右下或底部存在 `Bid`、`Start Action`、技能/道具装备等强操作按钮。
- 有货币总量显示，且数值量级很大，常见几十 K、几 M、几千万级。

### 3.3 角色/收藏家系统

视频帧显示角色页包含：

- 多名现实人物风格收藏家，例如 Chen Mei、Aisha、Gabriella、Tatiana、Victor 等。
- 左侧页签：Details、Appearance、Voice Lines。
- 右侧角色头像网格，部分带感叹号提示。
- 每个角色有职业/身份背景、技能名、技能效果说明。
- 有 `Collector Tasks`，说明角色任务和角色成长/解锁挂钩。
- 局前还可装备技能，部分技能或道具有数量限制。

### 3.4 藏品/市场/仓库系统

截图和公告显示其局外经济很厚：

- `Collection` 页面支持 Purchase、Sell、Transaction History。
- 左侧分类包括 Home & Daily Use、Medical Supplies、Fashion & Trends、Weapons & Gear、Minerals & Gems、Antiques & Curios、Digital & Electronics、Energy & Transportation、Food & Cooking 等。
- 商品卡包含英文名称、物品图、价格、稀有度颜色。
- `Store / Exchange` 页面含 Item Exchange、Ticket Exchange、Warehouse Box Exchange、Bidder Exchange、Gift Box Exchange。
- 官方公告提到 Daily Shop、Silver Coin Chest、Secret Auction、Express Blind Box、Auction House、Exchange、mailbox、warehouse、collection cabinet、guild filter、personal room item filtering 等。

### 3.5 玩家反馈画像

评论 API 抽样显示吸引点和痛点都很明确：

- 正向吸引点：题材新颖、梭哈刺激、赌性强、短时间上头、开出高价值物有爽点。
- 高频负反馈：服务器、爆率、充值/氪金、黑盒、红色藏品、脚本/外挂、算法诱导、拍不到仓、破产后难恢复。
- 这说明它真实运行中的玩家感受已经从“推理拍卖”偏向“刷宝赌博 + 经济系统信任危机”。

## 4. 当前 Demo 实现画像

本地核验：

- 配置校验通过：6 个角色、121 个藏品、30 个货柜。
- 当前主流程：4 人房间，1 名人类 + 3 个 AI 补位，5 轮脚本。
- 拍卖模式：明拍、暗标、次高价、押金明拍、闪拍。
- 角色：鉴定师、走私商、心理师、造谣者、修复师、保险商。
- 信息结构：公共估值、公共线索、私人线索、技能线索、假情报、推荐安全价。
- 结算结构：成交价、真值、修复费、押金、保险返还、套装加成、每名玩家分账、线索复盘。
- 局外：图鉴、任务、段位、收藏柜、战令与商城占位、排行榜占位。
- 后台：对局列表、逐轮回看、事件流、货币流水、历史复盘。
- 美术：已有背景、角色、货柜、藏品资产，当前 UI 走三国珍宝/动物掌柜风格。

## 5. 玩法差异表

| 模块 | 《竞拍之王》观察到的状态 | 当前 Demo | 差异判断 |
|---|---|---|---|
| 对局人数 | 4 人在线对局，支持在线 PvP/合作标签 | 4 人房间，AI 补位为主 | 框架接近，但真实多人压力不足 |
| 回合结构 | 同一仓库约 5 轮逐步揭示 | 固定 5 轮、每轮不同货柜 | 当前核心结构不符，必须先改成单仓五轮 |
| 拍卖形式 | 明拍/暗拍两类体验最关键 | 明拍、暗标、次高价、押金、闪拍并列 | 模式过散，应先打准明拍与暗拍 |
| 局前准备 | 角色和包装入口存在 | 选角色、创建房间、开始 | 当前阶段不是核心，不列 P0 |
| 侦查信息 | 平均值、稀有度、轮廓、品类扫描等技能 | 公共线索、私人线索、技能线索 | 概念接近，但呈现方式不同 |
| 开箱表现 | 格子化 Loot 区，稀有度框，逐步揭示 | 线性 RevealStrip 和 item chip | 视觉爽点明显弱于竞品 |
| 经济量级 | 几万到几千万，多货币，多交易 | 初始 100,000，局内短循环 | 属于周边经济，当前后置 |
| 市场交易 | Collection 买卖、Exchange、Auction House、邮箱 | 图鉴/收藏柜展示，无真实交易 | 周边系统，当前不作为核心差异 |
| 藏品分类 | 大量现实品类和市场商品分类 | 三国珍宝、古玩、军需、宫廷物 | 我们主题更统一，但广度较小 |
| 角色数量 | 视频约十几名收藏家，带外观/语音/任务 | 6 名动物拟人角色 | 角色深度不足，风格差异大 |
| 道具系统 | 技能和物品可装备，商店兑换道具 | 暂无局内道具栏，只有角色技能 | 缺关键操作层 |
| 成长系统 | 成就、任务、排行榜、收藏等级、交易门槛 | 本地图鉴、任务、段位、排行榜占位 | 只有雏形 |
| 信任/反作弊 | 官方公告显示脚本、DDOS、交易治理压力大 | 服务端权威、事件流水、后台回看 | 我们方向正确，但未做高压安全 |
| 复盘 | 官方公告提 replay 修复，具体复盘表达未知 | 结算复盘和后台事件流较强 | 这是我们当前优势 |

## 7. 当前最值得补的差异

### P0：核心玩法差异

1. 把对局模型改成“一个仓库跑 5 轮”，每轮不换货柜。
2. 建立单仓渐进揭示：轮廓、稀有度、品类、数量、估值区间、技能标记逐轮增加。
3. 明拍模式跑通 5 轮：公开价格、领先者、跟价/放弃、最终成交。
4. 暗拍模式跑通 5 轮：隐藏报价、轮后排名/差距/区间反馈、最终成交。
5. 把右侧仓库改成格子化 Loot 面板，作为局中推理界面，而不是只做开箱展示。
6. 角色技能必须改变玩家对同一仓库的可见信息，并进入每轮消息与复盘。

### P1：包装层差异

1. 普通仓、沉船仓、黑市仓、宫廷仓等只作为背景、藏品池、稀有度分布和风险文案的包装。
2. 角色外观、动物拟人或三国珍宝题材都属于包装，不影响核心规则。
3. 仓型可以影响数值分布，但不能引入门票、交易所、商城等周边系统。

### P2：界面表现差异

1. 角色选择页增强为全身立绘大屏，右侧角色网格，左侧技能/外观/语音/任务。
2. 局内 HUD 强化：玩家席位、出价状态、技能提示、货币变化、稀有度扫描结果更醒目。
3. 加快捷出价：上一轮出价、推荐倍率、键盘输入、常用数字按钮、自定义快捷价。
4. 出货反馈：稀有/传说藏品需要全屏动效、音效位、日志广播位。

## 8. 不建议照抄的部分

1. 不建议做黑箱动态爆率。竞品大量负评集中在“爆率/黑盒/算法诱导/官方故意让输”的信任问题。
2. 不建议早期上强充值闭环。当前设计目标是可上线内部 Demo，先验证公平和乐趣。
3. 不建议只堆角色和藏品数量。竞品负评也说明内容量不能替代可信结算和稳定服务器。
4. 不建议弱化复盘。竞品如果玩家认为“别人拍就出货，我拍就是垃圾”，复盘缺失会直接放大不信任。

## 9. 对当前产品方向的修正建议

当前 Demo 的正确方向是“更可信、更可解释的竞拍之王替代品”。下一轮迭代建议改为：

```text
单仓五轮运行时
→ 渐进揭示与玩家可见信息
→ 明拍五轮
→ 暗拍五轮
→ 角色技能信息差
→ 格子 Loot 局中推理界面
→ 最终开箱与每轮复盘
```

短期不要急着做全市场、充值、局前装备和活动仓。先把“玩家通过 5 轮信息和出价反馈推断最佳竞拍价格”做出来，这才是当前复刻目标。
