# 20260526 BidKing 策划归档 05 试宝令与局内道具

## 基础结论

`BattleItem` 有 64 行。运行层先按 `itemName_<BattleItemId>` 直接找 `Skill`，找不到才走 `SkillGroup` 兜底。当前 64 个都有结构化 `BattleItemEffectPlan`。

## 道具效果分布

| 类型 | 效果侧重 | 当前数量概览 |
| --- | --- | --- |
| type 1 | 藏品本体/完整信息 | 10 个，主要 revealKind=`identity` |
| type 2 | 轮廓侦察 | 14 个，主要 revealKind=`footprint` |
| type 3 | 总占格统计 | 7 个，revealKind=`quantity` |
| type 4 | 平均占格统计 | 5 个，revealKind=`quantity` |
| type 5 | 命中数量统计 | 6 个，revealKind=`quantity` |
| type 6 | 总价/价格 | 8 个，revealKind=`value` |
| type 7 | 品质/轮廓 | 9 个，主要 revealKind=`quality` |
| type 8 | 均价 | 5 个，revealKind=`value` |

## 使用限制

| 限制 | 规则 |
| --- | --- |
| 使用阶段 | 只能在 `intel` 或 `auction` |
| 竞价前限制 | 若已出价或已有本轮 bid，竞价阶段不能再用 |
| 每回合次数 | 默认 1 次，受系统技能 Category 21 修改 |
| 冷却 | 来自 `Skill.skill_CD`，每回合递减 |
| 目标玩家 | 若计划要求目标玩家，必须指定其他玩家 |
| 目标格 | 若 `skilltarget=8`，需要目标格 |
| 空命中 | 记录“没有命中”，不再随机兜底点亮 |

## 策划设计意义

试宝令是局内的主动情报资源，和竞买人的自动技能形成互补：

- 自动竞买人技能决定本局的信息底色。
- 试宝令决定玩家何时补关键情报。
- 竞价阶段的“出价前可用”让玩家能在最后提交前再做一次风险确认。
- 每回合次数和冷却是核心平衡杠杆。
