# 本地固定端口与服务脚本

## 固定地址

- Web：`http://127.0.0.1:5188`
- 管理后台：`http://127.0.0.1:5188/admin`
- API / Socket 服务：`http://127.0.0.1:8787`

## 脚本

在 `bitkingdom` 目录执行：

```powershell
npm run dev:local
```

纯本地前台启动 Web 与服务端，固定使用 `5188` 和 `8787`。端口被占用时不会自动换端口，会直接报错。

## 公网默认口径

后续运行、调试、验收和重启默认以公网地址为准：

```powershell
npm run dev
```

该脚本匹配当前映射：

- Web：本机 `5188`，公网 `https://kingdom-web.iepose.cn`
- API / Socket：本机 `5187`，公网 `https://kingdom-server.iepose.cn`

前端会通过 `VITE_SERVER_URL=https://kingdom-server.iepose.cn` 请求注册、登录和 Socket.IO 服务。公网脚本会先构建 Web，再用 `vite preview` 监听 `5188`，避免 Vite dev HMR 在公网转发下继续尝试连接 `localhost:5188`。

公网别名仍可使用：

```powershell
npm run dev:public
```

公网重启后必须确认：

- `5188` 和 `5187` 均处于监听状态。
- `https://kingdom-web.iepose.cn` 返回 `珍宝局 Demo`。
- `https://kingdom-server.iepose.cn/health` 返回 200。
- 注册请求走 `https://kingdom-server.iepose.cn/api/account/register` 并返回 200。
- 公网页面不再注入 `@vite/client`，控制台不应出现 `127.0.0.1:8787`、`localhost:5188` 或 Vite HMR WebSocket 报错。

```powershell
npm run dev:start
npm run dev:stop
npm run dev:restart
```

后台启动、停止、重启公网固定端口服务。`dev:stop` 会同时清理旧开发端口 `5177`、`8798`、`8787` 上残留的旧进程，避免访问到过期实例。

公网映射后台别名：

```powershell
npm run dev:public:start
npm run dev:public:stop
npm run dev:public:restart
```

纯本地后台脚本：

```powershell
npm run dev:local:start
npm run dev:local:stop
npm run dev:local:restart
```
