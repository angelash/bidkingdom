## 落账链路

原版 Unity 客户端大量状态集中在 `PlayerGameData`：

| 数据 | 来源 |
| --- | --- |
| 玩家货币和道具数量 | `GetItemNum()`、仓库/背包协议 |
| 角色拥有状态 | `heros` + 对应角色 `Item` + 体验卡 |
| 皮肤拥有状态 | 仓库中的 `Item_Type=14` |
| 当前房间 | `curRoomData` |
| 模拟训练 | `simGameLog` |
| 市场寄拍 | `sendAuctions` |
| 个人资料 | `profile` |
| 活动列表 | `nowActivityList` |
| 低保状态 | `subsistence` |

局内结算后，服务器应把获得藏品、利润、亏损、任务进度、排行、协会积分、收藏等级等写回这些局外状态。源码客户端中的局外面板都围绕这些状态刷新。

## 局外设计结论

竞拍之王的局外不是简单菜单。它的功能目标是：

- 让局内获得的藏品有收藏、交易、陈列、排行价值。
- 让角色、皮肤、体验卡和任务绑定长期成长。
- 让门票、道具、低保、商店控制局内尝试成本。
- 让市场、协会、排行把竞拍结果转成社交竞争。
