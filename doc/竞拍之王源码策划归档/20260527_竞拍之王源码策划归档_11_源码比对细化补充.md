## 终局揭露表现验收点

| 源码点 | 策划结论 |
| --- | --- |
| `Battle_Main.DoItemEffect()` | `itemDatas` 先全部 `GridItem.Build(data)`，再全部 `GridItem.DoSearchLoop()`，形成全仓库 loading 搜索态 |
| `items.Sort((a, b) => a.itemData.pos - b.itemData.pos)` | 揭露顺序按仓库 `pos` 从小到大，不按品质、价值或掉落顺序 |
| `GridItem.DoSearchLoop()` | 播放 `ani_search_loop`，设置 `canShowDetail=false`；品质、本体图标、名称和价格在搜索态不作为最终揭露结果出现 |
| `GridItem.DoEffect(speed)` | 按品质播放 `ani_bai/ani_lv/ani_lan/ani_zi/ani_jin/ani_hong/ani_caise`，等待结束后设置 `canShowDetail=true` |
| `MainUtils.PlayAnim()` | 从 Animator Controller 的 AnimationClip 查找同名 clip，播放并返回 clip length |
| `griditem.data` | 包含 `img_loading_1` loading 图素、`ani_search_loop` 循环动画和各品质揭露动画 |
| 局内仓库面板 | `Battle_Main.Init()` 使用 `WareHouseData.Build(40)` 建 10 列 40 行仓库；`Grid.Set(WareHouse)` 启用 `itembox_pic_1` 正式格子；仓库本体不展示结算摘要；`CalYuguPrice()` 保留 `yuguTxt` 当前预估最低价格，按 `BattleGridItemData.GetYujiPrice().x` 累加 |

| 品质 | 动画 | clip 时长 | 默认等待 |
| ---: | --- | ---: | ---: |
| 1 | `ani_bai` | 0.7 s | 1000 ms |
| 2 | `ani_lv` | 0.7 s | 1000 ms |
| 3 | `ani_lan` | 1.1 s | 1000 ms |
| 4 | `ani_zi` | 1.633 s | 2000 ms |
| 5 | `ani_jin` | 2.1 s | 2000 ms |
| 6 | `ani_hong` | 2.567 s | 3000 ms |
| 7 | `ani_caise` | 3.583 s | 4000 ms |

等待公式为 `floor(clip.length + 0.5) * 1000 / speed`。`ani_search_loop` 约 1.333 s 且循环播放。私人房间 `speed=2`，普通模式使用玩家本地开启速度设置。

## 局内情报列表与技能展示规则

| 源码点 | 策划结论 |
| --- | --- |
| `Battle_Handler.S2C_OnRoundStart(GameData)` | 每回合先筛 `MapSkillLog.CastRound == roundNum`，再筛 `HeroSkillLog.CastRound == roundNum`；普通局只有 `curRound == 0 && gameSkillDatas.Count > 0` 时打开 `IntelligencePanel` 并等待 `TacketTask`，后续回合不再播放暗牌情报，随后才逐条处理地图技能，再逐条处理角色技能 |
| `IntelligencePanel.PlayMaxTacketAni()` | 四张暗牌候选按约 `1f-1.9f` 随机时长轮选，停 `0.5s` 后调用 `IntelligenceItem.OnShowCard()`，明牌后停留，再播放关闭动画 |
| `Battle_Handler.DealSkillDataAsync()` | 每个 `GameSkillData` 先派发 `E_OnSkillTrigger`，非忽略技能等待 `CreateSkillTriggerTask()` 被 UI 触发后才继续处理下一个技能或同步仓库变化 |
| `Battle_Main.AddSkillDesc()` | `show_type=0` 时先 `Battle_SkillShow.Build(GameSkillData)` 弹出技能展示卡，再 `MoveTo()` 列表目标位；移动完成后实例化 `Battle_SkillDescItem` 到 `skillInfoRect.content`，然后 `TriggerSkillTask()` 放行下一条；`OnNewRound()` 不清空该列表，已出现信息累计保留 |
| `Battle_Main.DelayDoSkillDesc()` | `isShowingSkillDesc` 和 `caches` 队列保证技能展示串行执行，不并发刷入列表 |
| `Battle_SkillDescItem.Build()` | 列表行名称由来源名加技能名组成：角色取 `HeroCid`，场景取 `MapCid`，道具取 `ItemCid`；描述取 `Skill.skill_textshow`，为空才用 `skilldesc` |

因此局内中心情报列表的唯一内容源是技能日志流：`MapSkillLog -> HeroSkillLog -> ItemSkillLog`。`IntelligencePanel` 的暗牌结果是开局公共情报表现，只在首轮出现，不再作为一条独立“公共”记录重复插入同一列表；角色私有线索也不再作为“掌眼”记录和 `HeroSkillLog` 重复展示。可见顺序必须是先弹场景/地图技能卡，技能卡隐藏或移动完成后该行进入列表；再弹角色/道具技能卡，依次进入列表。后续回合追加新触发技能，列表保留此前已入列的信息。

## 复刻约束

- 玩法数值、流程内核、掉落、技能目标、信息披露和结算字段以源表和协议为准。
- `BattleRandom_Main` 和 `IntelligencePanel` 是源码 UI 流程，属于必须保留的表现层证据；普通/房间/回放只在匹配入口 `BidMap.IsRandomMap()` 为真时显示 `BattleRandom_Main`。
- 原版线上服务端不可见的内部算法按协议字段、客户端消费方式和源表含义推断实现；推断结果必须保持客户端可观测行为等价。
- 表字段缺失、引用缺失或未知枚举不产生替代规则，应按源表约束暴露错误。
