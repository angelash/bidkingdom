# 20260527 竞拍之王源码策划归档 11 - 源码比对细化补充

## 比对口径

本补充只使用 `reverse/bidking` 下的反编译 Unity 客户端、协议类、原始配置表，以及 `packages/bidking-compat/src/tables` 中由原表生成的 typed 数据。结论按“源码字段/方法/协议证据 -> 策划规则”归档。

| 证据层 | 代表路径 | 用途 |
| --- | --- | --- |
| 表类源码 | `reverse/bidking/code/decompiled/Scripts/Table_*.cs` | 字段名、加载列、索引方式、派生属性 |
| 协议类 | `reverse/bidking/code/decompiled/Scripts/Protodata/*.cs` | 服务端权威字段、局内/局外落账结构 |
| 客户端逻辑 | `Battle_Handler.cs`、`Battle_Main.cs`、`BattleRandom_Main.cs`、`IntelligencePanel.cs`、`PlayerGameData.cs` | UI 流程、事件顺序、协议消费方式 |
| generated 表 | `packages/bidking-compat/src/tables/*.ts` | 行数、分布、表间引用统计 |

## 复刻约束

- 玩法数值、流程内核、掉落、技能目标、信息披露和结算字段以源表和协议为准。
- `BattleRandom_Main` 和 `IntelligencePanel` 是源码 UI 流程，属于必须保留的表现层证据；普通/房间/回放只在匹配入口 `BidMap.IsRandomMap()` 为真时显示 `BattleRandom_Main`。
- 原版线上服务端不可见的内部算法按协议字段、客户端消费方式和源表含义推断实现；推断结果必须保持客户端可观测行为等价。
- 表字段缺失、引用缺失或未知枚举不产生替代规则，应按源表约束暴露错误。
