# 20260526 竞拍之王源码策划归档 02 - 原版场景与 BidMap

## 表关系

`Map` 是大场景入口，`BidMap` 是可进入的具体拍场。局内实际读取 `BidMap`：

| 字段 | 策划含义 |
| --- | --- |
| `parent_map_id` | 所属大场景，对应 `Map.id` |
| `map_group` | 随机拍场池。非空时会进入 `BattleRandom_Main` 随机实际 `BidMap` |
| `map_cell` | 拍场规模档，源 UI 作为场地格数/体量展示 |
| `currency_cost` | 入场货币消耗 |
| `item_cost` | 指定门票消耗 |
| `required_items` | 入场资产或物品要求 |
| `map_time` | 每轮倒计时，通常 40/50/60 秒 |
| `drop_group_id` | 掉落入口，格式近似 `[类型, dropId, min, max]` |
| `bidder_number` | 参与人数 |
| `auction_rounds_rate` | 每轮成交倍率，千分比 |
| `map_random_skill` | 源表按回合列出场景技能组 id；普通局可见场地情报复刻只取首轮开局一次 |
| `map_resource` | UI/Loading 图资源 |

源码表类 `Table_BidMap` 会把 `id` 建入 `ms_idMap`，同时按 `parent_map_id` 建入 `ms_parent_map_idMap`。`RandomMaps` 属性只在 `map_group` 至少包含一个有效 `[bidMapId, weight]` 条目时成立；空数组行表示普通子拍场，不进入随机父池。

## 成交倍率

常规拍场大多使用：

```text
[2000,1600,1300,1100,0]
```

按源码 Demo 解释为：

| 回合 | 倍率 | 含义 |
| ---: | ---: | --- |
| 1 | 2.0 | 最高价要高于第二价 2 倍才提前成交 |
| 2 | 1.6 | 成交门槛下降 |
| 3 | 1.3 | 进入中后段 |
| 4 | 1.1 | 接近摊牌 |
| 5 | 0 | 最终轮成交 |

训练地图 `parent=201` 使用 `[0,0,0,0,0]`，用于模拟而不是标准匹配成交线。

`GameServerDemo.ServerHandler.CheckGameOver()` 的判断是 `maxPrice > secondMax * rate`，其中 `rate = auction_rounds_rate[round - 1] * 0.001`。所以常规拍场前四轮可转写为最高价分别高出第二名 100%、60%、30%、10% 才提前成交，第五轮由服务端结算收口。

## 场景技能

`map_random_skill` 源表通常按五回合配置：

- 快递/仓库：`[102,103,103,104,105]`，源表五个槽位均填写概率池字段。
- 集装箱：`[103,0,103,0,103]`，隔轮触发。
- 别墅：`[103,0,103,0,0]`，后期场景信息更少。
- 沉船/隐秘拍卖会：`[103,0,0,0,0]`，主要依赖角色和局内道具。

具体技能从 `SkillGroup` 权重池抽取，例如 `101-105` 分别标记第一至第五回合。当前普通局可见流程的场地情报验收口径是首轮开局一次；第二轮起不再由客户端或复刻端主动补生成新的场地情报条目，局内列表只累计保留首轮场地条目并追加后续角色、道具触发信息。

## 场景随机表现

`BattleRandom_Main` 是源码明确存在的随机拍场表现流程：

| 源码点 | 行为 |
| --- | --- |
| `BattleRandom_Main.OnShowImp(args)` | 参数中接收最终 `BidMap.id`，加载最终场景名和图片 |
| `BattleRandom_Item.Build(bidmapId)` | 对候选项加载 `Table_BidMap` 名称和 `LoadBidMapImage` |
| `BattleRandom_Main.GetRandomMapIds()` | 普通/房间/回放读取匹配入口 `gameMatchMapId` 的 `RandomMaps`；训练模式按实际 `gameMapId.parent_map_id` 找同父子拍场 |
| 结果展示 | 候选列表包含父池候选，最终高亮服务器给出的实际 `MapId` |

因此随机动画不是独立玩法阶段，也不是客户端决定实际拍场。实际 `BidMap` 由服务端 `GameData.MapId` 决定；普通/房间/回放只有匹配入口 `BidMap.IsRandomMap()` 为真时显示 `BattleRandom_Main`，该界面只负责把父拍场随机池和最终结果表现出来。
