# 20260526 竞拍之王源码策划归档 02 - 原版场景与 BidMap

## 表关系

`Map` 是大场景入口，`BidMap` 是可进入的具体拍场。局内实际读取 `BidMap`：

| 字段 | 策划含义 |
| --- | --- |
| `parent_map_id` | 所属大场景，对应 `Map.id` |
| `map_group` | 随机拍场池。非空时会进入 `BattleRandom_Main` 随机实际 `BidMap` |
| `map_cell` | 拍场规模档，源 UI 作为场地格数/体量展示 |
| `currency_cost` | 入场货币消耗 |
| `item_cost` | 指定门票消耗 |
| `required_items` | 入场资产或物品要求 |
| `map_time` | 每轮倒计时，通常 40/50/60 秒 |
| `drop_group_id` | 掉落入口，格式近似 `[类型, dropId, min, max]` |
| `bidder_number` | 参与人数 |
| `auction_rounds_rate` | 每轮成交倍率，千分比 |
| `map_random_skill` | 每回合场景技能组 id |
| `map_resource` | UI/Loading 图资源 |

## 成交倍率

常规拍场大多使用：

```text
[2000,1600,1300,1100,0]
```

按源码 Demo 解释为：

| 回合 | 倍率 | 含义 |
| ---: | ---: | --- |
| 1 | 2.0 | 最高价要高于第二价 2 倍才提前成交 |
| 2 | 1.6 | 成交门槛下降 |
| 3 | 1.3 | 进入中后段 |
| 4 | 1.1 | 接近摊牌 |
| 5 | 0 | 兜底最终轮 |

训练地图 `parent=201` 使用 `[0,0,0,0,0]`，用于模拟而不是标准匹配成交线。

## 场景技能

`map_random_skill` 通常按五回合配置：

- 快递/仓库：`[102,103,103,104,105]`，每轮都有概率池。
- 集装箱：`[103,0,103,0,103]`，隔轮触发。
- 别墅：`[103,0,103,0,0]`，后期场景信息更少。
- 沉船/隐秘拍卖会：`[103,0,0,0,0]`，主要依赖角色和局内道具。

具体技能从 `SkillGroup` 权重池抽取，例如 `101-105` 分别标记第一至第五回合。
