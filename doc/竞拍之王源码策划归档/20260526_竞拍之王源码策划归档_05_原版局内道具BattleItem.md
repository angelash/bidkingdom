# 20260526 竞拍之王源码策划归档 05 - 原版局内道具 BattleItem

## 总览

`BattleItem` 表 64 行，全部对应 `Item_Type=11` 的消耗型道具。局内道具不是独立技能表，而是：

```text
BattleItem.id -> Item.id -> Item.skills[0] -> Skill.id -> SkillEffect
```

源码中 `BattleItemData` 负责把普通对局、训练对局、房间对局里的已选择道具转成局内可用列表；`Battle_Handler.C2S_UseItem()` 根据模式发送普通、训练或房间协议。

## 使用限制

| 限制 | 源码表现 |
| --- | --- |
| 每轮通常只能用一次 | `checkCanUseItem()` 检查当前 `UseItemLog` 是否已有记录 |
| 已弃权不可使用 | Demo 服务端检查 `playerQiquanDic` |
| 已出价不可使用 | Demo 服务端检查 `playerAuctions` |
| 道具消耗后立即触发技能 | 普通对局走 `S2C_39_game_use_item`，训练/房间直接补放新增 `ItemSkillLog` |
| 训练道具有 buff 道具和选择道具两类 | `SimBuffItemList` 与 `SimSelectItemList` |

普通局 `C2S_38_game_use_item`、房间局 `C2S_186_room_game_use_item`、训练局 `C2S_128_sim_game_use_item` 都以 `ItemCid` 为核心输入。服务器返回的不是“道具成功”布尔值，而是新增 `ItemSkillLog`；客户端把它并入局内日志后播放对应技能结果。

| Category | 数量 | 道具方向 |

| 4 | 6 | 数量统计 |

## 道具与策略

- 轮廓类道具帮助判断仓库中大件或稀有件，但不一定直接给价值。
- 品质类道具更适合判断是否值得加价，尤其与某些角色的类型信息叠加。
- 价值统计类道具最接近“可出价上限”，强度高、品质高、成本也高。
- 完整抽样类道具适合在局势不确定时找锚点，但随机性仍然存在。
- 均价/格数类道具服务于估算，不直接翻牌，更考验玩家理解。
