# 20260526 竞拍之王源码策划归档 07 - 解锁门槛经济与限制

## 入场门槛

原版拍场进入前至少校验三层：

| 层级 | 配置/源码 | 说明 |
| --- | --- | --- |
| 货币和门票消耗 | `BidMap.currency_cost`、`BidMap.item_cost` | 进入拍场要扣金币或指定票 |
| 资产/物品要求 | `BidMap.required_items` | 常见为金币总资产要求，如 10000、100000、500000、2000000、5000000、10000000 |
| 仓库容量 | `MainUtils.CheckSlotItemIsFull()` | 仓库满时不可进入竞拍 |

`BattlePrevPanel.EnterBattle()` 会先校验费用，再校验资产要求，再校验仓库容量。

## 门票体系

`Item_Type=2` 是功能道具(门票)。原表中有：

| Item.id | 名称 | 用途 |
| ---: | --- | --- |
| 101 | 竞拍票 | 基础竞拍入场消耗 |
| 102 | 指定快递竞拍票 | 快递子拍场 |
| 103 | 指定仓库竞拍票 | 仓库子拍场 |
| 104 | 指定集装箱竞拍票 | 集装箱子拍场 |
| 105 | 指定别墅拍票 | 别墅子拍场 |
| 106 | 指定沉船竞拍票 | 沉船子拍场 |

`Ticket` 表只有 1 行，描述基础竞拍票恢复和购买策略；具体拍场消耗主要落在 `BidMap.item_cost`。

## 低保限制

原版有低保机制，战前金币/资产不足时会提示：

| 常量 | 值 | 含义 |
| --- | --- | --- |
| `relief_fund_times` | `3` | 每日领取次数 |
| `relief_fund_limit` | `100000` | 当前总资产低于该值时可领 |
| `relief_fund_amount` | `[[1,1,100000]]` | 发放 100000 金币 |

源码注释明确总资产包含普通货币总价、仓库所有藏品价值和收藏柜藏品价值。

## 交易和拍卖行限制

`Constant` 中交易相关配置：

| 常量 | 含义 |
| --- | --- |
| `item_fee_rate` | 道具上架手续费，千分比 |
| `item_bid_fax` | 拍卖税 |
| `item_bid_fee` | 拍卖保管费 |
| `item_bid_price_add` | 拍卖加价增量，万分比 |
| `item_bid_time` | 可选拍卖周期，`[24,48,72]` 小时 |
| `item_bid_cost` | 24 小时上架费，按起拍价万分比 |
| `item_bid_public_time` | 拍卖公示期，24 小时 |
| `item_bid_slot_base` | 基础上架栏位 5 |
| `item_bid_slot_max` | 上架栏位上限 10 |
| `auction_time_limit` | 临近结束有人出价时恢复倒计时 |
| `auction_slot_price` | 购买栏位价格 |
| `auction_counts` | 出价次数上限 |
| `auction_price_notice` | 高价购买提示倍率 |
| `bid_lv` | 收藏等级达到 25 才可用交易行 |

## 亏损返利

`bid_fanli=[10000,100]` 表示结算时如果亏损超过 10000，会按千分比参数给参与玩家返还部分亏损，弃权玩家不算。这是原版对高风险竞拍的保护阀。

## 任务和条件

`Condition` 表 206 行，`Mission` 表 314 行，`Achievement` 表 25 行。常见条件类型包括：

| condition | 示例 | 用途 |
| ---: | --- | --- |
| 6 | 在竞拍中累计使用道具 N 次 | 道具使用任务、成就 |
| 7 | 竞拍成功后获得 N 件藏品 | 结算获得物任务 |
| 9 | 交易所购买 N 件 | 市场买入任务 |
| 10 | 交易所出售 N 件 | 市场卖出任务 |
| 12 | 解锁角色/拥有皮肤 | 角色成长任务 |
| 14 | 在竞拍中出价达到阈值 | 出价档位成就 |
| 15 | 参与指定场所竞拍 | 参与次数任务 |
| 16 | 指定角色竞拍成功 | 角色专属任务 |
| 17 | 竞拍失败次数 | 失败相关任务 |
| 18 | 单次竞拍净利润 | 高收益成就 |
| 19 | 获得指定价值藏品 | 结算价值成就 |
| 20 | 登录天数 | 登录任务 |
| 24 | 累计净利润 | 长线收益成长 |
| 25 | 当前总资产低于阈值 | 低保触发 |

任务和成就不是局内硬规则，但它们驱动角色、资源、道具、称号、通行证和长期目标。

## 房间自定义边界

私人房间允许配置：

- 模式：暗拍为主，传统拍卖注释为未实装。
- 竞拍轮数：5/10/15/20。
- 地图：101-105 大场景。
- 初始点数、道具预算、每轮时间、人数。
- 可选角色和可选局内道具。
- 自定义成交倍率。

因此原版不是只有公开匹配，还有强社交/房间规则口径。
