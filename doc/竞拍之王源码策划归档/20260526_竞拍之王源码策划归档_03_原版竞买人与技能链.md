# 20260526 竞拍之王源码策划归档 03 - 原版竞买人与技能链

## 总体规则

原版共有 20 个竞买人。配置入口是 `Hero` 表：

- `Hero.id` 是角色唯一 id。
- `name / hero_class / background / hero_skill_name / hero_effect_description` 都是本地化 key，中文来自 `Language.txt`。
- `cast_type` 是局内技能链。数组下标对应回合顺序，非 0 项指向 `Skill.id`。
- `access=[2,380]` 是原表获取字段，但源码实际拥有状态还会看 `HeroData`、角色物品和体验卡。
- `voices` 指向 `Sound` 表，`battleBgm` 指向战斗 BGM。

## 技能链类型

| 类型 | 代表角色 | 策划作用 |

| 类型专项全揭示 | 陈美、塔蒂安娜、乔治、伊万 | 快速锁定某一类高价值品类 |

| 聚合统计 | 娜奥米、玛丽亚、艾哈迈德、维克托 | 不直接翻格，而给总价、数量、平均格数等宏观线索 |
| 极值检索 | 伊莎贝拉、法蒂玛部分技能 | 指向最高品质或最高价值藏品 |
| 空间线索 | 伊森 | 以轮廓和已知品质联动，帮助判断仓库布局 |

## 拥有与体验

源码中的角色拥有状态不是只看 `Hero.access`。`PlayerGameData.GetHeroState()` 逻辑是：

- `RealHave`：服务端 `HeroData` 中有角色，且玩家仓库拥有对应角色物品。
- `XianMian`：服务端 `HeroData` 中有角色，但没有对应角色物品，可理解为限免/临时授权状态。
- `TiYan`：玩家拥有 `Item_Type=19` 的体验卡，且体验卡 `skills` 包含该 `Hero.id`。
- `NotHave`：以上都没有。

角色物品位于 `Item_Type=15`，体验卡位于 `Item_Type=19`，每个角色都有对应体验卡，例如 `8101` 法蒂玛体验卡、`8102` 陈美体验卡等。
