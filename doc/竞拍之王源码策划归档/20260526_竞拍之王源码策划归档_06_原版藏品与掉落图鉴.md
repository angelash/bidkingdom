## Item 表定位

`Item` 表 1136 行，是原版最核心的综合物品表。它包括：

- 货币和基础资源。
- 门票、仓库盒、收藏柜、宝箱。
- 局内消耗道具。
- 竞买人角色、皮肤、体验卡。
- 10 类核心藏品。
- 特殊藏品、头像、称号、表情、成就勋章。

核心字段：

| 字段 | 含义 |
| --- | --- |
| `item_name / item_desc` | 本地化 key |
| `item_type_id` | 类型数组，可有多标签 |
| `slot_type` | 占格尺寸，十位是宽，个位是高 |
| `item_quality` | 品质，1-6 为常规藏品品质，6 也用于部分特殊物 |
| `base_value` | 基础卖店价/估值，源码 `SalePrice` 直接取它 |
| `is_tradable` | 是否可交易 |
| `auction_baseprice` | 拍卖起价配置 |
| `transaction_tax_rate` | 交易税配置 |
| `max_stack_size / max_per_listing` | 堆叠和上架限制 |
| `skills` | 道具或角色对应技能 |
| `drop_group_id` | 使用/开启时的掉落组 |
| `icon_path / icon_atlas` | 图标资源 |
| `collection` | 收藏相关数值 |
| `rank7count` | 彩色编号/稀有编号相关 |
| `collection_coin` | 收藏柜收益基础 |
| `number / number_weight` | 编号池和权重 |
| `model_3D` | 3D 模型占位 |

## 掉落生成

`BidMap.drop_group_id` 是局内生成入口。源码 Demo 解释为：

```text
drop_group_id = [type, id, min, max]
count = RandomCount(min, max)
if type == 9999:
  id.DoDrop(count)
else:
  直接加入 id * count
```

`Drop.items_list` 中每项近似为：

```text
[type, itemIdOrDropId, min, max, weight]
```

当 `type == 9999` 时继续递归到另一个掉落组，否则落到具体 `Item.id`。因此一个拍场不是直接列出所有藏品，而是通过多层掉落组按品质、品类和权重生成本局仓库。

## 占格与仓库

`Table_Item.Size` 由 `slot_type` 计算：

```text
Size.x = slot_type / 10
Size.y = slot_type % 10
SizeCount = Size.x * Size.y
```

这意味着技能里的“轮廓”“总格数”“平均格数”“单位格价值”都依赖 `slot_type`。原版技能不是只看 Item 文案，而是把藏品作为一个可摆放在仓库网格中的实体。
