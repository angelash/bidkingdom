# 20260526 BidKing 策划归档 08 AI 与 Bot

## RankAi 规模

`RankAi` 共 120 行，覆盖 20 个 Hero，每个 Hero 6 行，对应回合 1-6。

## Bot 行为树

Bot 决策由行为树驱动：

1. 情报阶段先判断是否用试宝令。
2. 非核心旧技能在核心局被禁用，核心局主要依赖自动 Hero 技能和试宝令。
3. 竞价阶段在未出价前也可用试宝令。
4. 最后根据公开估值、技能可见信息、风险、上一轮排名、成交线和 `RankAi` 参数决定出价或停手。

## 关键 AI 参数

| 字段 | 用途 |
| --- | --- |
| `min_bid_ratio` | 最低出价倾向区间 |
| `bid_pk` | 对抗/压价强度 |
| `item_use_probability` | 使用试宝令概率，当前分布 0-700 |
| `item_usage_group` | 试宝令掉落组 |
| `bid_time` | Bot 延迟/节奏 |
| `risk_appetite` | 风险偏好 |
| `bluff_chance` | 诈唬倾向 |
| `overpay_tolerance` | 溢价容忍度 |
| `bid_aggression` | 进攻性 |

## 策划理解

Bot 不是只按真实价值出价，而是综合以下信息形成可解释行为：

- 公共估值中点和估值宽度。
- 私有技能线索与已显示格位的覆盖程度。
- 当前拍场风险。
- 自身上一轮出价与排名。
- 当前回合成交线压力。
- `RankAi` 给出的最低出价、对抗比例和道具使用偏好。

后台事件会记录 `BotActionAudit`，可复盘 Bot 估值、信心、目标价、最高可出价、成交线、动作类型和试宝令选择。
