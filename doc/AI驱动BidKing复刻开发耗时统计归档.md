## 统计字段说明

Codex 会话中可直接读取以下字段：

| 字段/事件 | 含义 | 本次用途 |
| --- | --- | --- |
| `task_started` | 一轮用户任务开始 | 确定 turn 起点 |
| `task_complete.duration_ms` | 一轮任务从开始到完成的耗时 | 作为“Codex 任务耗时累计”的主口径 |
| `task_complete.time_to_first_token_ms` | 从任务开始到第一次输出的时间 | 观察响应启动延迟 |
| `response_item.reasoning` | 思考事件 | 内容加密，不读取正文；耗时已包含在 `duration_ms` 中 |
| `function_call / function_call_output` | Shell、搜索、读取文件等工具调用 | 估算工具等待和命令执行时间 |
| `custom_tool_call / output` | `apply_patch` 等自定义工具调用 | 估算文件编辑和补丁应用耗时 |
| `image_generation_call` | 图像生成任务 | 估算素材流水线中 AI 生图相关耗时 |

因此，`duration_ms` 是本次最硬的统计依据。它包含模型思考、工具执行、等待命令、读取文件、写补丁和组织最终回复，但不代表单个人坐在屏幕前连续操作的时间。

## 工作类型判定口径

本次耗时按“任务主目标”归类，不按单个文件类型归类。例如某轮任务既改代码又补文档，如果用户目标是修正一条玩法流程，就归入“核心玩法与局内流程实现”；如果目标是沉淀方案、归档规则或写复盘，就归入“文档、方法论与规则维护”。

| 工作类型 | 判定标准 | 典型任务 |
| --- | --- | --- |
| 文档、方法论与规则维护 | 目标是沉淀规则、整理证据、统一口径或维护长期文档 | 当前/原版策划分卷、AI 生产归档、MDC 规则、仓库维护规则、耗时统计 |
| 核心玩法与局内流程实现 | 目标是让对局机制、状态流、局内 UI 或结算更接近原版 | 同仓五轮、仓库揭示、竞买人技能、出价限制、轮选随机、结算 loading |
| 素材与包装流水线 | 目标是生产、替换、命名或校验自制视觉/文案包装 | 藏品图生成、正式美术批次、珍宝命名、角色包装文案、素材清单校验 |
| 逆向取证与原版规则分析 | 目标是从原版源码、表、资源或截图视频确认事实 | Unity 文件分析、配置字段含义、客户端/服务端职责、Item 分类 |
| 局外系统、UI 与性能优化 | 目标是局外页面、展示、首屏资源或非核心对局 UI 表现 | 珍阁、市场、货币展示、首屏下载量、局外页面表现 |
| 运维支持与服务操作 | 目标是让服务、端口、忽略目录或环境可继续工作 | 启动服务、端口排查、公网访问、目录忽略 |
| 其他相关任务 | 与 BidKingdom 有关，但难以稳定归入单一类型 | 跨主题排查、临时说明、多个目标混在一起且主目标不清晰的会话 |

## 墙上时间口径

由于 Codex 可以同时存在多个会话，不能简单把所有会话首尾跨度相加。这里给出两种墙上时间：

| 口径 | 说明 | 结果 |

| 完成 turn 区间直接去重 | 每轮按 `task_started -> task_complete` 形成区间，重叠区间只算一次 | 约 94.9 小时 |
| 相邻任务 5 分钟内合并 | 允许短暂停顿、上下文切换和命令收尾 | 约 97.6 小时 |
| 相邻任务 30 分钟内合并 | 视为同一开发工作段，适合估算高强度工作时间 | 约 109.8 小时 |
| Codex 任务耗时累计 | 并行会话不去重，反映 AI 任务执行总量 | 约 127.8 小时 |

推荐解释：

- 对外说“累计 AI 协作任务时间”时，用 128 小时。
- 对外说“实际高强度开发时间”时，用 95-110 小时。
- 如果强调人机并行生产能力，可以同时说明：部分素材生产和长任务并行，因此累计任务时间大于墙上时间。

## 可信度和限制

本统计可信度较高的部分：

- Codex `task_complete.duration_ms` 是直接记录，不是人工估算。
- 会话索引和 JSONL 可复查线程名称、用户任务和时间戳。

仍需注意的限制：

- `reasoning` 内容加密，不能读取具体思考正文；但思考耗时已包含在 `duration_ms` 内。
- 工具调用耗时只能按调用和输出时间戳估算，部分流式工具或长任务可能有微小偏差。
- 多个会话并行时，“累计任务耗时”和“墙上时间”天然不同，不能混用。
- 分类依赖任务标题、用户请求和关键词，复杂任务可能跨多个类型。
- 本统计不计算用户离线思考、手动验收截图、人工审美判断和未进入 Codex 的工作时间。
