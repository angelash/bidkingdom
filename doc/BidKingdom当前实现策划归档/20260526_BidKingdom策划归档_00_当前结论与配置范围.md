# 20260526 BidKingdom 策划归档 00 当前结论与配置范围

## 当前结论

当前 BidKingdom 工程已经把核心配置抽成 `@bitkingdom/bidking-compat` typed 表，共 52 张表、19687 行。与核心玩法直接相关的表包括拍场、藏品、竞买人、技能、局内道具、AI、任务、交易、UI、声音、本地化和系统限制。

当前可运行核心体验是：玩家从主界面选择拍场和竞买人，进入房间，按 `BidMap` 人数补 Bot，创建同一批隐藏仓库，进行多轮暗拍/明拍，回合开始自动触发拍场技能和竞买人技能，玩家可在情报/竞价前使用试宝令，竞价后按 `BidMap.auction_rounds_rate` 判断是否成交，最终揭示藏品并结算收益、图鉴、仓库、任务、成就和协会积分。

需要特别标记的源码现状：当前 `packages/match-core/src/match.ts` 和 `apps/server/src/domain/battle/roomRoundRuntime.ts` 仍保留首轮 `warehouse_roll -> warehouse_selected -> auctioneer_reveal` 三段候选仓/唱牌官流程；但 `doc/20260525_BidKing旧Demo逻辑残留归档.md` 里写过“已删除”。本批归档以当前源码为准。

## 复刻目标口径

BidKingdom 的策划目标是原版《竞拍之王》玩法 100% 还原。数值、流程内核、匹配/房间规则、拍场选择、仓库生成、成交线、技能触发、技能目标、道具效果、Bot 行为参数、结算落账、任务条件和经济限制都必须以 `reverse/bidking` 原版源码、原始配置表和客户端协议行为为准。

三国/古代化只属于表现层包装：可替换角色名、称号、剧情文案、美术资源、音频资源、UI 皮肤和展示描述，但不得改变任何 `Hero.cast_type`、`Skill`、`SkillEffect`、`BattleItem`、`BidMap`、`Drop`、`RankAi`、`Condition` 等表驱动的游戏性含义。

因此，本组文档中记录的“当前源码状态”不是默认设计定稿。凡与原版源码口径不一致的现状，都应进入偏差清单并按原版还原；只有文案和美术包装差异可被接受。
