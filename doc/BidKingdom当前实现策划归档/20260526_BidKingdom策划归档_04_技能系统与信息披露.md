# 20260526 BidKingdom 策划归档 04 技能系统与信息披露

## 技能目标选择

`Skill.skilltarget / skilltarget2 / skilltarget3` 通过 `skillTargeting.ts` 解释：

| target | 含义 | 例子 |
| ---: | --- | --- |
| 0 | 随机抽样 | 随机显示 N 件 |
| 1 | 按藏品类型 | 101 府邸日用、106 文玩古器 |
| 2 | 按品质 | 1 到 6 |
| 3 | 按具体 Item.id | 指定某件藏品 |
| 4 | 加权随机类型 | 从配置类型池里抽若干类型 |
| 5 | 加权随机品质 | 从品质池里抽 |
| 6 | 排序/极值筛选 | 最高价值、最低价值、最大格数、同类数量等 |
| 7 | 按占格面积 | 例如只命中 4 格以上 |
| 8 | 指定仓库格 | 需要玩家点选目标格 |
| 10 | 按已知状态 | 轮廓未知、品质未知、完整信息未知 |

`skill_count=0` 按全量命中处理；`skill_count_type=2` 表示按比例命中，例如 3333 约等于三分之一。

| 17 | 修改金币利息上限 | 系统效果 | 修改 `gameGoldRateMax` |

## 情报可见性规则

| 信息 | 限制 |
| --- | --- |
| 估值区间 | `estimateHidden=true` 时客户端收到 0/0，不直接展示内部总估值；前端只按已知格子/线索汇总“已知情报估算”，对应原版 `Battle_Main.CalYuguPrice()` |
| 地图技能 | 公共；核心局从 `intel` 开始即可显示 |
| 竞买人技能 | 私有，仅技能拥有者可见 |
| 试宝令 | 私有，仅使用者可见 |
| hitBoxList | 按 Category 做字段遮罩，不能因为命中格而泄漏完整藏品 |
| 仓库格 | 初始以 1x1 未知格显示；只有技能/道具命中的字段会逐步变真 |
| 出价金额 | 暗拍中隐藏；明拍在反馈后可见；最终结算后历史出价可见 |
| 拍场情报暗牌 | 作为地图公共情报进入 `intel`，按原版 `IntelligencePanel` 表现为四张暗牌随机揭示 |

## GameSkillData 日志字段

| 字段 | 来源 | 实现 |
| --- | --- | --- |
| `skillCid` | `Skill.id` | `skillFeedToGameSkillLog()`、`battleItemEventSkillLogs()` |
| `heroCid` | 触发玩家映射到的 `Hero.id` | 英雄技能和试宝令日志必填；地图技能为 0 |
| `mapCid` | 实际 `BidMap` | 地图技能日志写入；其他来源为 0 |
| `itemCid` | `BattleItem.id` | 试宝令日志写入；其他来源为 0 |
| `castTime` | 事件时间 | `SkillFeedEntry.createdAt` 或 `MatchEventLog.createdAt` |
| `castRound` | 回合序号 | `round.index + 1` 或事件所在回合 |
| `hitBoxList` | 命中仓库格 | `bidKingSourceHitBoxList()` 和 `boxInfoForSlot()` |
| `hitItemIndex` | 命中件数 | 命中 slot 数 |
| `allHitItemAvgPrice` | 命中总价 / 件数 | 无命中为 0 |
| `allHitBoxAvgPrice` | 命中总价 / 格数 | 无格数为 0 |
| `allHitItemAvgBoxIndex` | 命中总格数 / 件数 | 无命中为 0 |
| `hitItemTotalPrice` | 命中总价 | 命中 slot 的 `item.value` 合计 |
| `totalHitBoxIndex` | 命中总占格 | `w * h` 合计 |
| `hitItemTypeList` | 命中品类集合 | 源 Item 类型去重 |
| `hitItemQuilityList` | 命中品质集合 | 源 Item 品质去重 |

日志生成入口固定为 `buildSkillLogs()`。英雄、地图和试宝令分别进入 `heroSkillLog`、`mapSkillLog`、`itemSkillLog`；试宝令使用事件日志补齐，避免漏掉使用后生成的 `ItemSkillLog`。

## 情报表现分层

| 表现层 | 数据源 | 作用 |
| --- | --- | --- |
| `BattleRandom_Main` | `openingCandidates` | 仅匹配入口 `BidMap.map_group` 非空时展示候选拍场轮选和最终实际拍场 |
| `IntelligencePanel` | `intelligenceChoices`、`intelligenceClue` | 每轮展示四张暗牌随机揭示一条地图公共情报 |
| `BattleIntelligence_Main/2/3` | `publicClues`、`privateClues`、`skillFeed` | 承接地图技能、竞买人技能和试宝令的可见情报列表 |

上述表现只消费 `intel` 内的数据，不创建额外玩法阶段，也不改写技能命中、道具次数、出价窗口和计时。
