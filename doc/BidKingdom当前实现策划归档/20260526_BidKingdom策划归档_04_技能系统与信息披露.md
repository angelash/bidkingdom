# 20260526 BidKingdom 策划归档 04 技能系统与信息披露

## 技能目标选择

`Skill.skilltarget / skilltarget2 / skilltarget3` 通过 `skillTargeting.ts` 解释：

| target | 含义 | 例子 |
| ---: | --- | --- |
| 0 | 随机抽样 | 随机显示 N 件 |
| 1 | 按藏品类型 | 101 府邸日用、106 文玩古器 |
| 2 | 按品质 | 1 到 6 |
| 3 | 按具体 Item.id | 指定某件藏品 |
| 4 | 加权随机类型 | 从配置类型池里抽若干类型 |
| 5 | 加权随机品质 | 从品质池里抽 |
| 6 | 排序/极值筛选 | 最高价值、最低价值、最大格数、同类数量等 |
| 7 | 按占格面积 | 例如只命中 4 格以上 |
| 8 | 指定仓库格 | 需要玩家点选目标格 |
| 10 | 按已知状态 | 轮廓未知、品质未知、完整信息未知 |

`skill_count=0` 按全量命中处理；`skill_count_type=2` 表示按比例命中，例如 3333 约等于三分之一。

命中顺序以原表目标类型和技能文案共同决定。`skilltarget=0` 直接随机抽样；`skilltarget=4/5` 先随机抽类型或品质，再按结果筛选；`skilltarget=6` 使用原版极值排序和 itemId、uid tie-break，不额外打乱；`skilltarget=8` 固定命中指定格。固定数量筛选技能只有源表文案明确写“随机”的技能才在筛选后用本局 RNG 抽样，例如法蒂玛 `1001011-1001014`、加布里埃拉 `1001041-1001045`、索菲 `1001071-1001074`、卡洛斯 `1002021-1002024`、武田宏志 `1002062-1002065`，以及吴起灵 `10002073` 的 `target=1 + countType=2,count=3333`。海琳娜 `1001091-1001094` 虽然也是 `target=1 + target=10`，但源表文案没有“随机”，保持原版筛选顺序取前 N 个。`skill_count=0` 全量命中时不需要打乱。

## 情报可见性规则

| 信息 | 限制 |
| --- | --- |
| 估值区间 | `estimateHidden=true` 时客户端收到 0/0，不直接展示内部总估值；局内保留原版 `Battle_Main.CalYuguPrice()` 的 `yuguTxt` 当前预估最低价格，按已同步格子的确定价、具体藏品、轮廓尺寸、占格数和品质计算，不把品类文本作为估价过滤条件 |
| 地图技能 | 公共；只在首轮开局生成一次场地情报，第二轮起不再新增 `source: map` 信息 |
| 竞买人技能 | 私有，仅技能拥有者可见 |
| 试宝令 | 私有，仅使用者可见 |
| hitBoxList | 按 Category 做字段遮罩，不能因为命中格而泄漏完整藏品 |
| 仓库格 | 初始以 1x1 未知格显示；只有技能/道具命中的字段会逐步变真 |
| 出价金额 | 暗拍中隐藏；明拍在反馈后可见；最终结算后历史出价可见 |
| 拍场情报暗牌 | 仅普通局首轮作为地图公共情报进入 `intel`，按原版 `IntelligencePanel` 表现为四张暗牌随机揭示，明牌后保留阅读停顿；第二轮起不再生成 `intelligenceChoices/intelligenceClue` |

## GameSkillData 日志字段

| 字段 | 来源 | 实现 |
| --- | --- | --- |
| `skillCid` | `Skill.id` | `skillFeedToGameSkillLog()`、`battleItemEventSkillLogs()` |
| `heroCid` | 触发玩家映射到的 `Hero.id` | 英雄技能和试宝令日志必填；地图技能为 0 |
| `mapCid` | 实际 `BidMap` | 首轮地图技能日志写入；其他来源为 0 |
| `itemCid` | `BattleItem.id` | 试宝令日志写入；其他来源为 0 |
| `castTime` | 事件时间 | `SkillFeedEntry.createdAt` 或 `MatchEventLog.createdAt` |
| `castRound` | 回合序号 | `round.index + 1` 或事件所在回合 |
| `hitBoxList` | 命中仓库格 | `bidKingSourceHitBoxList()` 和 `boxInfoForSlot()` |
| `hitItemIndex` | 命中件数 | 命中 slot 数 |
| `allHitItemAvgPrice` | 命中总价 / 件数 | 无命中为 0 |
| `allHitBoxAvgPrice` | 命中总价 / 格数 | 无格数为 0 |
| `allHitItemAvgBoxIndex` | 命中总格数 / 件数 | 无命中为 0 |
| `hitItemTotalPrice` | 命中总价 | 命中 slot 的 `item.value` 合计 |
| `totalHitBoxIndex` | 命中总占格 | `w * h` 合计 |
| `hitItemTypeList` | 命中品类集合 | 源 Item 类型去重 |
| `hitItemQuilityList` | 命中品质集合 | 源 Item 品质去重 |

日志生成入口固定为 `buildSkillLogs()`。英雄、地图和试宝令分别进入 `heroSkillLog`、`mapSkillLog`、`itemSkillLog`；试宝令使用事件日志补齐，避免漏掉使用后生成的 `ItemSkillLog`。

## 情报表现分层

| 表现层 | 数据源 | 作用 |
| --- | --- | --- |
| `BattleRandom_Main` | `openingCandidates` | 仅匹配入口 `BidMap.map_group` 非空时展示候选拍场轮选和最终实际拍场 |
| `IntelligencePanel` | `intelligenceChoices`、`intelligenceClue` | 普通局首轮展示四张暗牌随机揭示一条地图公共情报，明牌后留出阅读停顿 |
| `BattleIntelligence_Main/2/3` | `skillFeed` | 承接地图技能、竞买人技能和试宝令的可见情报列表；列表不重复插入暗牌公共情报或角色私有线索，且按已出现信息累计展示 |

上述表现只消费 `intel` 内的数据，不创建额外玩法阶段，也不改写技能命中和出价规则。中心情报列表按 `MapSkillLog -> HeroSkillLog -> ItemSkillLog` 串行展示：`MapSkillLog` 只在首轮新增一次，已出现条目常驻保留；第二轮起只对本轮新增的名士/道具技能先弹技能提示卡，提示卡隐藏或移动完成后该条记录进入列表，再处理下一条。右侧仓库格信息与该串行节奏绑定：`match-core` 在 `intel` 阶段不把当前轮技能预先应用到公共 `warehouseSlots`，当前轮技能只通过 `skillFeed.hitBoxList` 下发；未落入列表的本轮技能不参与仓库显示，落入列表的同一帧才由前端按该条 `hitBoxList` 同步揭示其命中的轮廓、品质、价值或完整藏品字段。当前端本地提示序列未完成时，即使公共阶段已经进入 `auction`，也继续按隐藏格、历史日志和已落当前日志重建右仓，避免完整 `warehouseSlots` 提前泄漏。
