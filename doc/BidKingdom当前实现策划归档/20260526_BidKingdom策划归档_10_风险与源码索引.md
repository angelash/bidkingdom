# 20260526 BidKingdom 策划归档 10 风险与源码索引

## 玩法还原验收原则

当前项目目标是 100% 还原原版《竞拍之王》的游戏性内容。任何数值、流程内核、技能链、技能目标、道具效果、掉落、AI、结算、任务、经济限制和局外落账差异，都应按原版源码和原始配置表修正。三国/古代化只允许作为文案、美术、音频和 UI 视觉包装存在。

若原版线上后端没有源码参考，则按协议推断实现，不作为阻塞项。推断优先级为：原始配置表和表类、Unity 客户端协议字段与 UI 消费行为、协议快照日志结构、旧 Demo 可验证公式、当前项目为补齐后端权威而写的最小等价逻辑。所有协议推断实现都要在报告中标注，后续拿到更强证据时再替换。

## 当前实现边界与风险

| 风险 | 当前源码状态 | 建议 |
| --- | --- | --- |
| 文档与源码不一致 | 旧归档说首轮旧阶段已删除，但当前源码仍有三段首轮流程 | 按原版 `Battle_Handler` / `GameData` / UI 流程复核；若无原版证据，应删掉或降级为纯表现动画，不得改变状态机和规则 |
| `gameConfig.roles` 残留 3 个未绑定角色 | UI 已过滤，但原 config 仍存在旧角色 | 若不是原版 `Hero` 映射或纯表现包装，应迁移到过时配置，避免参与任何玩法逻辑 |
| `Hero.packaged_name` 与原版姓名不同 | 当前运行用三国/古代包装名，原版实名见 20260525 竞买人归档 | 可保留为包装名，但 `Hero.id`、`cast_type`、职业/技能链、获取条件和数值含义必须逐项对齐原版 |
| 1136 行 Item 不宜复制进策划文档 | 本批文件只提炼分类和核心字段，完整行在 generated 表 | 若策划需要逐件校对，另导出 CSV/表格 |
| 系统类 SkillEffect 多为结构化解释 | Category 16-28 已能形成 operation，但不是每类都有完整局内表现 | 逐项按原版 `SkillEffect`、协议日志和对应 UI 表现补齐，不允许自定义效果替代 |
| 公共估值隐藏 | 内部生成估值，但客户端隐藏为 0/0 | 按原版 `Battle_Main`、`GameData.StockContainer` 和信息披露 UI 复核；展示与隐藏口径必须和原版一致 |
| 当前 Bot 为本地行为树 | 已读取 `RankAi`，但存在本地启发式估值和决策 | 以原表 `RankAi` 字段语义和原版服务端/客户端可见行为为准；服务端不可见部分按协议推断实现并标注 |
| 成交线解释需锁定 | 当前实现按领先差距判断，文档中也引用旧 Demo `maxPrice > secondMax * rate` | 以原版 `BidMap.auction_rounds_rate` 和可验证服务端/Demo 行为统一验收，测试覆盖每轮成交、未成交、平价和最终轮 |
