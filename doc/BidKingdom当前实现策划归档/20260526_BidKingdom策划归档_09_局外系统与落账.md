# 20260526 BidKingdom 策划归档 09 局外系统与落账

## 档案初始化

新档案默认：

| 项 | 值/来源 |
| --- | --- |
| 初始金币 | `Constant.init_items` 中 `[1,1,2000000]`，即 2000000 |
| 初始头像 | `Constant.init_head=120000` |
| 默认选中 Hero | `bidKingDefaultHeroId()`，由初始拥有/试用角色决定 |
| 票券 | `Ticket[0]`，竞拍票上限 20，恢复时间 0 |
| 初始物品 | `Constant.init_items`，含藏品、角色、体验卡、表情、宝箱等 |

## 结算落账

最终结算后：

| 流程 | 行为 |
| --- | --- |
| 支付 | 赢家扣除成交价 |
| 获得藏品 | 赢家获得整仓藏品，进入库存和实体仓库 |
| 金币盈亏 | 对局 `auctionStats.netProfit` 作为 signed 金币流水写入 `match_reward_coins`，盈利加铜钱、亏损扣铜钱 |
| 图鉴 | 新藏品点亮 codex；重复藏品按 `cabinet_rate` 给经验 |
| 等级 | `LevelUp.collection_value` 决定等级 |
| 亏损返利 | 若亏损达到 `bid_fanli[0]`，按 `bid_fanli[1]` 千分比返还 |
| 任务/成就 | 刷新 Mission/Achievement/Condition |
| 协会积分 | 若有协会，按 `GuildPoints.profit` 区间给积分 |
| 仓库/收藏柜 | 实体藏品用 `slot_type` 占格，可入柜产生收益 |

## 局外窗口

`UIWnd` 共 80 个窗口，前端把核心入口映射到：

| 模块 | 标题 | 源窗口 |
| --- | --- | --- |
| 图鉴 | 珍宝谱 | `HandBook_Main` |
| 任务 | 委托 | `Task_Main` |
| 排行 | 名士榜 | `Rank_Main` |
| 收藏柜/仓库 | 珍阁 / 行囊 | `PackagePanel` |
| 市集/拍卖行 | 市集 / 拍场 | `TradingExchange_Main` / `AuctionPlacePanel_Main` |
| 商店 | 宝铺 | `StorePanel` |
| 协会 | 鉴宝会 | `Party_Main` |
| 活动/通行证 | 珍宝令 | `BattlePass_Main` |
| 好友/邮件 | 同游 / 信札 | `Friend_Main` / `Mail_Main` |

`UIWnd` runtime 会解释 `IsMainWnd`、`Layer`、`IsBlur`、`BGM`、`ResSet`，生成 `main/modal/overlay/panel` 和关闭行为。

## 货币展示

BidKingdom 前端货币展示统一使用中制紧凑单位：`10000` 起显示为 `万`，`100000000` 起显示为 `亿`，例如初始 `2000000` 铜钱在大厅钱包和战前资产处显示为 `200万`，不再使用 `K`。金额不足 `10000` 时保留千分位数字。

## 送拍协议

| 流程 | 源字段/协议 | 实现 |
| --- | --- | --- |
| 创建委托 | `SendAuctionData.Uid/MapCid/SlotId/StockData/SendTime` | `createSendAuctionForProfile()` 校验地图、件数、估价、手续费和仓库来源，`buildSourceSendAuctionData()` 输出快照，`SendTime` 为 Unix 秒 |
| 成交 | `SendAuctionGameData.GameData/UserSkillList/GameOverTime` | `settleSendAuctionForProfile()` 生成 `SendAuctionGameState`，按 `RankAi` 生成虚拟竞买人出价日志，`GameOverTime` 为 Unix 秒 |
| 回收 | `SendAuctionRecycleResponse.ErrorCode` | `recycleSendAuctionForProfile()` 生成回收结果和邮件 |
| 列表 | `SendAuctionListSnapshot`、`SendAuctionGameListSnapshot` | `buildSendAuctionListSnapshot()`、`buildSendAuctionGameListSnapshot()` |

送拍用同一套 `BidKingGameDataSnapshot`。`gameData.stockContainer` 来自委托锁定藏品，`gameData.userLog.priceLog` 记录虚拟竞买人出价，`sendAuctionUserUid/sendAuctionUserName` 指向委托玩家，`gameData.nextRoundTime/serverTime` 与源协议一致使用 Unix 秒。

## 邮件与商店协议

| 数据 | 源字段 | 实现 |
| --- | --- | --- |
| `MailItemData` | `ItemCid`、`ItemCount`、`ItemNo`、`ItemQuility` | `mailAttachmentRewards()` 解析 `Mail` 表奖励行，`claimMailForProfile()` 调用 `applyRewardRowsToProfile()` 落账 |
| 自定义邮件 | 标题、正文、附件奖励行 | `addCustomMailToProfile()` 过滤敏感词并生成 `MailInboxItem.attachmentRewards` |
| 邮件容量 | `Constant` 邮箱上限 | `bidKingMailMaxCount()` |
| `ShopStatusData` | `ShopCid`、`NextRefreshTime`、`ShopItemList` | `BidKingShopStatusDataSnapshot`、`ShopRestockState`、`ShopPurchaseState` |
| 商店刷新 | `Shop.autofresh`、随机商品池 | `refreshShopRestock()`、`refreshExpiredShopRestocks()` |
| 商店购买 | `ShopItem.front/price/itemid/buycounts` | `buyShopItemForProfile()` 校验解锁、货币、材料和购买上限 |
