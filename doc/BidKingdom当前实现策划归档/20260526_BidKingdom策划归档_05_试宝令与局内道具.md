# 20260526 BidKingdom 策划归档 05 试宝令与局内道具

## 基础结论

`BattleItem` 有 64 行。局内道具技能链固定为 `BattleItem.id -> Item.id -> Item.skills[0] -> Skill.id -> SkillEffect`。缺少 `Item`、`Item.skills`、`Skill` 或 `SkillEffect` 时直接报错；不走 `SkillGroup` 补链。64 个道具都有结构化 `BattleItemEffectPlan`。

## 道具效果分布

| 类型 | 效果侧重 | 数量概览 |
| --- | --- | --- |
| type 1 | 藏品本体/完整信息 | 10 个，主要 revealKind=`identity` |
| type 2 | 轮廓侦察 | 14 个，主要 revealKind=`footprint` |
| type 3 | 总占格统计 | 7 个，revealKind=`quantity` |
| type 4 | 平均占格统计 | 5 个，revealKind=`quantity` |
| type 5 | 命中数量统计 | 6 个，revealKind=`quantity` |
| type 6 | 总价/价格 | 8 个，revealKind=`value` |
| type 7 | 品质/轮廓 | 9 个，主要 revealKind=`quality` |
| type 8 | 均价 | 5 个，revealKind=`value` |

## 使用限制

| 限制 | 规则 |
| --- | --- |
| 使用阶段 | 只能在 `intel` 或 `auction` |
| 竞价前限制 | 若已出价或已有本轮 bid，竞价阶段不能再用 |
| 每回合次数 | 默认 1 次，受系统技能 Category 21 修改 |
| 冷却 | 来自 `Skill.skill_CD`，每回合递减 |
| 目标玩家 | 若计划要求目标玩家，必须指定其他玩家 |
| 目标格 | 若 `skilltarget=8`，需要目标格 |
| 空命中 | 记录“没有命中”，不随机点亮其他格 |

## 运行字段

| 字段/函数 | 含义 |
| --- | --- |
| `BattleItemEffectPlan.itemId` | 源 `BattleItem.id` |
| `skillGroupId` | 源 `BattleItem.skill_group`，只作为源字段记录 |
| `skillId/skillName` | `Item.skills[0]` 对应的 `Skill` |
| `effectId/effectCategory/effectKey` | `Skill.skilleffect_position[0]` 对应的 `SkillEffect` |
| `requestedTargetCount` | `Skill` 原始目标数量 |
| `targetCount` | 实际命中数量 |
| `durationRounds` | `Skill.skill_round` |
| `cooldownRounds` | `Skill.skill_CD` |
| `revealKind` | `value/category/quality/quantity/footprint/identity/system` |
| `targetMode` | `skill_target/highest_value/largest_slots/system_effect` |
| `implementationStatus` | 仓库情报为 `implemented`，系统类效果为 `protocol_inferred` |

`useBattleItem()` 负责生成私有线索、写入 `round.skillFeed`、设置冷却并推送 `battle_item_used` 事件。`buildSkillLogs()` 再把该事件转换成 `GameData.itemSkillLog`，字段保持 `S2C_39_game_use_item` 的 `ItemSkillLog` 语义。

## 实现落点

| 环节 | 文件 | 规则 |
| --- | --- | --- |

## 策划设计意义

试宝令是局内的主动情报资源，和竞买人的自动技能形成互补：

- 自动竞买人技能决定本局的信息底色。
- 试宝令决定玩家何时补关键情报。
- 竞价阶段的“出价前可用”让玩家能在最后提交前再做一次风险确认。
- 每回合次数和冷却是核心平衡杠杆。
