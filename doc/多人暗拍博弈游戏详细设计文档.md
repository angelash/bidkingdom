# 多人暗拍博弈游戏详细设计文档

> 暂定项目名：《珍宝局》 / 《暗拍大师》 / 《盲拍奇兵》  
> 文档版本：V0.1  
> 产品方向：4人轻竞技拍卖博弈 + 信息差 + 心理战 + 收藏成长  
> 目标平台：微信小游戏 / 移动端优先，可扩展至 Steam  
> 单局时长：6～10分钟  
> 核心原则：公平竞技、短局强反转、结果可信、付费不影响局内胜负

---

## 1. 项目定位

### 1.1 一句话定位

一款4人同局的轻竞技暗拍游戏。玩家通过公共线索、私人情报和角色技能判断货柜价值，在明拍、暗标、次高价拍卖等模式中出价、诈唬、诱导他人接盘，最终以局内净资产排名决定胜负。

### 1.2 核心体验

玩家的核心体验不是“单纯竞价”，而是：

- 看线索，判断货柜真实价值；
- 用表情、假动作、出价节奏误导对手；
- 低价捡漏时获得强爽感；
- 高价坑到别人时获得社交爽感；
- 被坑后能通过复盘知道自己错在哪里；
- 每局都有开箱反转和收藏成长反馈。

### 1.3 产品关键词

- 多人博弈
- 信息不对称
- 暗拍竞价
- 心理战
- 捡漏
- 互坑
- 开箱揭晓
- 收藏图鉴
- 公平竞技
- 短局轻竞技

### 1.4 竞品问题规避原则

同类产品容易出现以下问题，本项目从底层设计规避：

| 竞品常见问题 | 本项目原则 |
|---|---|
| 服务器不稳导致货币被吞 | 服务端权威结算，所有货币流水可追溯 |
| 赢得拍卖反而经常亏 | 引入次高价拍卖、线索保底、亏损保护机制 |
| 付费角色/本金导致P2W | 付费不影响局内初始资金、情报强度和胜负能力 |
| 玩家觉得系统黑箱 | 局后展示线索、估值、成交价、收益和判断依据 |
| 新手不知道什么时候该放弃 | 新手局提供估值区间、推荐出价和低风险机器人池 |
| 正反馈少 | 每局提供图鉴、任务、精彩回放、称号和成长奖励 |

## 2. 目标用户与平台

### 2.1 目标用户

#### 主目标用户

- 喜欢轻竞技、派对、博弈、桌游类玩法的玩家；
- 喜欢“互坑”“心理战”“猜价格”“开箱反转”的用户；
- 适合短视频传播和主播互动的社交型玩家；
- 微信小游戏内追求短局、低学习成本、强反馈的用户。

#### 次目标用户

- 喜欢收藏、图鉴、角色皮肤的轻度玩家；
- 喜欢经营收益、买低卖高、概率判断的策略玩家；
- 喜欢休闲竞技但不想高操作的用户。

### 2.2 平台适配

#### 微信小游戏版本

- 单局控制在6～8分钟；
- 操作以点击、滑动出价、快速表情为主；
- 首屏加载要快；
- 局外成长轻量化；
- 强调好友邀请、群分享、排行榜。

#### Steam / PC版本

- 可增加语音、房间、自定义局、观战和主播模式；
- 单局可扩展到10～15分钟；
- 可增加更复杂的角色、藏品和拍卖规则。

## 3. 核心设计目标

### 3.1 设计目标一：让玩家觉得“输得明白”

拍卖博弈类游戏中，玩家会频繁亏损。如果亏损原因不清楚，就会认为系统黑箱或数值骗人。

因此每局结算必须展示：

- 拍卖品真实价值；
- 成交价；
- 玩家利润；
- 本轮公共线索；
- 玩家获得的私人线索；
- 线索命中与误导点；
- 其他玩家关键出价行为。

### 3.2 设计目标二：让玩家觉得“赢得刺激”

赢的反馈不能只是数字上涨。

需要有：

- 开箱动画；
- 稀有藏品特写；
- 图鉴点亮；
- 对手表情反馈；
- 本轮称号，例如“极限捡漏”“血赚大师”“成功诱导”；
- 局后精彩回放。

### 3.3 设计目标三：付费不能破坏局内公平

玩家局内初始资金一致。任何付费内容不得提供：

- 额外本金；
- 更强情报；
- 更高藏品概率；
- 局内胜率加成；
- 专属强力技能；
- 更低竞价成本。

可付费内容包括：

- 角色外观；
- 拍卖桌皮肤；
- 表情；
- 入场动画；
- 收藏柜外观；
- 称号特效；
- 头像框；
- 非竞技模式娱乐规则包。

## 4. 核心玩法循环

### 4.1 局内循环

```text
匹配入场
→ 展示货柜
→ 获取线索
→ 玩家判断价值
→ 竞价/暗标/抬价
→ 成交
→ 开箱揭晓
→ 计算收益
→ 进入下一轮
→ 最终排名结算
```

### 4.2 局外循环

```text
完成对局
→ 获得经验/金币/图鉴进度/任务进度
→ 解锁角色/外观/表情/收藏柜
→ 提升段位或休闲积分
→ 再次匹配
```

### 4.3 长线循环

```text
收集藏品图鉴
→ 完成套装
→ 解锁收藏展示
→ 获得称号/皮肤/赛季奖励
→ 参与更高段位或活动玩法
```

## 5. 单局规则设计

### 5.1 基础局配置

| 项目 | 配置 |
|---|---|
| 玩家人数 | 4人 |
| 单局轮数 | 默认5轮，可扩展为7轮、10轮 |
| 初始资金 | 100,000局内币 |
| 单轮时间 | 60～90秒 |
| 单局总时长 | 6～10分钟 |
| 胜负目标 | 最终净资产最高者获胜 |
| 净资产计算 | 剩余资金 + 已购藏品估值 + 套装加成 - 惩罚费用 |

### 5.2 单轮流程

#### 5.2.1 货柜展示阶段

时间：8～12秒

展示内容：

- 货柜外观；
- 来源标签；
- 品类倾向；
- 初步估值范围；
- 风险提示；
- 是否可能含有套装藏品。

示例：

```text
货柜来源：破产古玩店
品类倾向：瓷器 / 字画 / 杂物
公共估值：20,000～80,000
风险提示：可能存在赝品
特殊提示：有小概率出现套装藏品
```

#### 5.2.2 情报阶段

时间：15～20秒

每名玩家可以执行一个情报行为：

- 使用角色技能；
- 查看私人线索；
- 投放误导信息；
- 观察对手状态；
- 放弃情报，获得少量资金补偿。

#### 5.2.3 竞价阶段

时间：25～35秒

根据本轮拍卖模式执行：

- 明拍；
- 暗标；
- 次高价拍卖；
- 押金拍卖；
- 闪拍。

#### 5.2.4 成交阶段

时间：3～5秒

展示：

- 成交玩家；
- 成交价格；
- 其他玩家最高出价或出价区间；
- 是否触发特殊规则。

#### 5.2.5 开箱阶段

时间：10～15秒

逐步揭示：

- 普通物品；
- 高价值物；
- 赝品；
- 破损品；
- 套装物；
- 隐藏稀有物。

开箱节奏要有层次：

```text
低价值杂物 → 中价值藏品 → 风险物 → 稀有物 → 最终估值
```

#### 5.2.6 单轮结算阶段

时间：8～10秒

展示：

- 成交价；
- 真实估值；
- 修复费/保管费/赝品损失；
- 最终利润；
- 本轮称号；
- 排名变化。

## 6. 拍卖模式设计

### 6.1 明拍模式

#### 规则

玩家轮流或自由加价，所有人可见当前最高价。

#### 特点

- 最容易理解；
- 适合新手；
- 适合制造抬价和诈唬；
- 容易出现上头过度出价。

#### 设计要点

- 设置最低加价单位；
- 设置倒计时延长机制；
- 连续加价次数过多时给出风险提示；
- 新手期显示推荐停手价。

### 6.2 暗标模式

#### 规则

所有玩家同时输入一个价格，倒计时结束后最高价者成交。

#### 特点

- 信息不透明；
- 心理判断强；
- 适合高手局；
- 容易产生“猜高一点”的紧张感。

#### 设计要点

- 玩家可看到其他玩家“出价意愿”而不是具体价格；
- 可显示表情、动作、犹豫时间；
- 支持角色技能探测某人的出价区间。

### 6.3 次高价拍卖模式

#### 规则

最高出价者获得货柜，但实际支付价格为第二高出价 + 固定加价单位。

示例：

| 玩家 | 出价 |
|---|---:|
| A | 80,000 |
| B | 65,000 |
| C | 50,000 |
| D | 30,000 |

A获胜，但只支付66,000。

#### 特点

- 降低过度出价带来的挫败；
- 鼓励玩家报出真实心理价；
- 对策略判断更友好；
- 适合作为核心主模式。

#### 设计要点

- 需要明确说明规则；
- 结算时展示第二高价来源；
- 防止玩家恶意极高报价，可设置资金上限和风险提示。

### 6.4 押金拍卖模式

#### 规则

参与竞价需要支付少量押金，最终未成交玩家返还部分押金或损失少量押金。

#### 特点

- 防止无成本乱抬价；
- 增加出价压力；
- 适合中后期局。

#### 风险

押金过高会降低参与感。建议押金只占货柜公共估值下限的3%～5%。

### 6.5 闪拍模式

#### 规则

展示信息更少，倒计时更短，成交后可能获得更高倍率收益。

#### 特点

- 节奏变化；
- 高风险高收益；
- 适合最后一轮逆转。

#### 设计要点

- 不要频繁出现；
- 建议只在第4～5轮出现；
- 新手场减少出现概率。

## 7. 货柜与藏品系统

### 7.1 货柜基础结构

每个货柜由多个藏品组成。

```text
货柜 = 来源标签 + 品类倾向 + 风险标签 + 若干藏品 + 隐藏修正
```

### 7.2 货柜标签

| 标签 | 说明 | 影响 |
|---|---|---|
| 古玩店清仓 | 藏品多，真假混杂 | 中高波动 |
| 豪宅遗物 | 高价值概率较高 | 高竞争 |
| 破产仓库 | 杂物多，偶尔出稀有 | 高波动 |
| 沉船打捞 | 破损概率高 | 需修复成本 |
| 军需旧库 | 武器、盔甲类较多 | 套装概率高 |
| 宫廷旧藏 | 稀有文物概率高 | 估值上限高 |
| 民间杂货 | 稳定低价值 | 适合新手 |

### 7.3 藏品品质

| 品质 | 颜色建议 | 价值区间 | 出现定位 |
|---|---|---:|---|
| 破烂 | 灰 | 100～2,000 | 填充、陷阱 |
| 普通 | 白 | 2,000～8,000 | 稳定收益 |
| 精良 | 蓝 | 8,000～20,000 | 常规惊喜 |
| 稀有 | 紫 | 20,000～60,000 | 主要爽点 |
| 传说 | 金 | 60,000～200,000 | 大反转 |
| 赝品 | 红/裂纹 | 估值虚高，结算低 | 误导和风险 |

### 7.4 藏品属性

每个藏品包含：

```text
藏品ID
名称
品类
品质
真实价值
表面价值
真假状态
破损状态
套装ID
稀有度权重
展示图标
开箱动画等级
```

### 7.5 赝品机制

赝品是重要的风险来源。

示例：

```text
表面价值：80,000
真实价值：5,000
线索提示：釉色异常 / 落款模糊 / 来源可疑
```

设计原则：

- 赝品不能完全无解；
- 至少有一个线索能提示风险；
- 高段位可提高赝品伪装度；
- 新手局降低赝品惩罚。

### 7.6 破损机制

破损藏品有修复费用。

```text
最终收益 = 真实价值 - 成交价 - 修复费
```

修复费建议占真实价值的10%～40%。

修复师类角色可降低修复费。

### 7.7 套装机制

多个藏品组成套装后获得额外价值。

示例：

| 套装 | 组成 | 加成 |
|---|---|---|
| 五虎兵器 | 5件兵器残件 | 总价值+50% |
| 宫廷玉器 | 3件玉器 | 总价值+30% |
| 古战场遗物 | 4件军需品 | 总价值+40% |

套装机制用于长线策略，不能让单局过度复杂。MVP阶段只做3～5套。

## 8. 信息与线索系统

### 8.1 信息类型

| 类型 | 可见范围 | 准确度 | 示例 |
|---|---|---:|---|
| 公共信息 | 所有人 | 100% | 货柜来自古玩店 |
| 半公开信息 | 所有人 | 70%～90% | 拍卖师称可能有瓷器 |
| 私人线索 | 单个玩家 | 60%～90% | 你看到一个金属反光物 |
| 技能信息 | 技能拥有者 | 依角色决定 | 总价值可能在5万～9万 |
| 假情报 | 受影响玩家 | 0%～50% | 有人散布里面有玉玺 |

### 8.2 线索品质

| 线索等级 | 准确度 | 说明 |
|---|---:|---|
| 模糊线索 | 50%～60% | 只提示大方向 |
| 普通线索 | 65%～75% | 可辅助判断 |
| 精准线索 | 80%～90% | 强判断依据 |
| 反向线索 | 特殊 | 可能被技能或假情报扭曲 |

### 8.3 线索示例

#### 价值线索

```text
你感觉这个货柜总价值不会低于40,000。
```

#### 风险线索

```text
有一件看似昂贵的瓷器釉色异常，可能是赝品。
```

#### 品类线索

```text
你看到货柜角落露出金属护甲的一部分。
```

#### 套装线索

```text
其中一件物品似乎属于“五虎兵器”系列。
```

#### 对手线索

```text
玩家B在看到线索后出价意愿明显提高。
```

### 8.4 线索生成规则

每轮至少生成：

- 1条公共信息；
- 1条风险提示或品类提示；
- 每名玩家1条私人线索；
- 若有角色技能，则追加技能线索。

### 8.5 线索复盘

局后必须展示：

| 线索 | 实际结果 | 判断价值 |
|---|---|---|
| 来源为古玩店 | 确实有瓷器和字画 | 有效 |
| 釉色异常 | 目标瓷器为赝品 | 命中 |
| 金属反光 | 实际为普通铜盆 | 误导 |

复盘的作用是降低玩家挫败，让玩家觉得“下次我可以判断得更好”。

## 9. 角色系统

### 9.1 角色定位

角色提供不同的信息获取方式和竞价风格，但不提供直接数值碾压。

### 9.2 角色设计原则

- 每个角色只有1个主动技能 + 1个轻量被动；
- 技能必须有冷却或使用次数；
- 技能不能直接获得完整答案；
- 技能不能直接增加局内初始资金；
- 所有角色都可通过免费方式获得；
- 付费只提供外观和表现。

### 9.3 角色示例

#### 角色1：鉴定师

定位：稳定估值

主动技能：价值鉴定

```text
查看当前货柜总价值的大致区间。
准确度：较高
冷却：2轮
```

被动：识假经验

```text
赝品相关线索准确度提高10%。
```

弱点：只能看总价值，无法判断价值集中在哪件物品，也容易被高修复费影响。

#### 角色2：走私商

定位：高价值单品判断

主动技能：盯货

```text
查看当前货柜中最高价值单品的大致品质。
```

被动：黑市眼光

```text
稀有品类线索出现概率略高。
```

弱点：如果货柜里只有一件高价值物但其他都是废品，可能仍然整体亏损。

#### 角色3：心理师

定位：观察对手

主动技能：读心

```text
查看一名玩家当前出价意愿区间。
```

被动：表情分析

```text
更容易发现对手是否使用了误导类技能。
```

弱点：不直接知道货柜价值，依赖对手行为。

#### 角色4：造谣者

定位：误导对手

主动技能：放风

```text
向所有玩家投放一条模糊假情报，假情报不一定完全错误，但会扭曲判断方向。
```

被动：话术熟练

```text
假情报被识破概率降低。
```

弱点：高段位玩家可通过复盘和反向判断识破。

#### 角色5：修复师

定位：降低破损风险

主动技能：预估修复

```text
查看当前货柜中是否存在高修复费物品。
```

被动：修复折扣

```text
自己拍下的破损藏品修复费降低20%。
```

弱点：面对赝品无优势。

#### 角色6：保险商

定位：新手友好、防暴亏

主动技能：风险保险

```text
本轮若亏损超过一定数值，返还部分损失。
```

被动：止损意识

```text
系统推荐停手价更加精准。
```

弱点：上限不高，适合稳健玩家。

#### 角色7：收藏家

定位：套装收益

主动技能：套装感应

```text
判断当前货柜是否包含套装藏品。
```

被动：收藏溢价

```text
套装完成时额外获得少量价值加成。
```

弱点：前期收益慢，容易被别人抢关键件。

#### 角色8：拍卖师

定位：竞价操作

主动技能：压秒叫价

```text
在明拍模式中可延迟一次出价，并触发短暂倒计时延长。
```

被动：加价节奏

```text
最低加价单位略低，便于试探。
```

弱点：对暗标模式帮助较小。

## 11. 经济与结算系统

### 11.1 局内经济

局内币只在单局内使用，局外不累计，避免付费影响竞技。

#### 初始资金

默认：100,000。

#### 出价限制

- 出价不得超过当前剩余资金；
- 可设置最低保留资金，例如最后一轮前不能低于5,000；
- 新手期提供风险提示。

### 11.2 局内净资产

```text
净资产 = 剩余资金 + 已购藏品真实价值 + 套装加成 - 修复费 - 保管费 - 罚金
```

### 11.3 单轮利润

```text
单轮利润 = 藏品真实价值 + 套装增益 - 成交价 - 修复费 - 风险费用
```

### 11.4 最终排名

| 排名 | 奖励 |
|---|---|
| 第1名 | 段位积分最多、经验最多、宝箱进度最多 |
| 第2名 | 中等奖励 |
| 第3名 | 少量奖励 |
| 第4名 | 保底奖励，新手期不扣或少扣 |

### 11.5 防负反馈机制

#### 新手保护

前10局：

- 提供推荐出价；
- 降低赝品概率；
- 降低修复费惩罚；
- 优先匹配机器人；
- 即使第4名也给图鉴和经验奖励。

#### 亏损保护

连续多局亏损时：

- 提高教学提示频率；
- 推荐保险商角色；
- 匹配较低强度池；
- 发放非竞技补偿奖励。

#### 单局暴亏限制

单轮最大亏损可设置软上限，特别是新手段位。

## 12. 局外成长系统

### 12.1 玩家等级

通过对局获得经验。

等级解锁：

- 新角色；
- 新表情；
- 新货柜主题；
- 收藏柜展示位；
- 非竞技模式。

### 12.2 图鉴系统

玩家拍到或开出藏品后点亮图鉴。

图鉴维度：

- 品类；
- 品质；
- 来源；
- 套装；
- 稀有事件。

图鉴奖励：

- 金币；
- 外观碎片；
- 称号；
- 收藏柜装饰；
- 角色皮肤折扣券。

### 12.3 段位系统

段位只影响竞技匹配和赛季奖励。

建议段位：

```text
新手 → 学徒 → 行家 → 鉴宝师 → 大掌柜 → 拍卖王 → 传说藏家
```

积分变化由排名决定，不直接由局内赚取金额决定，以降低高波动导致的挫败。

### 12.4 任务系统

#### 每日任务

- 完成3局；
- 成功拍下一件稀有藏品；
- 一局内至少一次盈利；
- 使用一次角色技能；
- 完成一次图鉴点亮。

#### 成就任务

- 极限捡漏：成交价低于真实价值30%以上；
- 反向大师：识破一次假情报；
- 赝品终结者：成功避开高价值赝品；
- 诱导成功：让对手以超过真实价值的价格成交；
- 套装收藏家：完成一个套装。

机器人不能太蠢，也不能像外挂。

| 类型 | 行为 |

- 机器人不会连续精准压制玩家；

- 教学提示增加；

## 19. 美术与包装方向

### 19.1 现代货柜方向

关键词：仓库、古玩、黑市、拍卖厅、神秘货柜。

优点：

- 用户理解成本低；
- 接近现实拍卖节目；
- 适合直播传播。

缺点：

- 同质化风险较高；
- 需要避免过于写实导致成本高。

### 19.2 三国珍宝方向

关键词：乱世、商队、军需、古董、兵器、玉玺、竹简、名将遗物。

优点：

- 可与三国题材资产协同；
- 品类天然丰富；
- 容易做套装和图鉴。

缺点：

- 需要避免让玩家误以为是传统三国卡牌；
- 题材包装要轻，不要压过玩法。

### 19.3 动物拟人拍卖方向

关键词：禽鸟拟人、动物商会、奇珍异宝、轻松诙谐。

优点：

- 角色记忆点强；
- 外观商业化空间大；
- 与轻竞技调性匹配。

缺点：

- 世界观需要统一；
- 藏品和角色风格要形成体系。

### 19.4 推荐方向

优先推荐：

```text
三国珍宝 + 动物拟人拍卖场
```

理由：

- 和已有三国/禽鸟拟人方向可复用；
- 题材差异明显；
- 适合做轻松互坑；
- 可扩展角色、皮肤、藏品、套装；
- 小程序用户更容易接受轻包装。

## 21. 新手引导设计

### 21.1 第一局教学目标

第一局只教会三件事：

1. 货柜有真实价值，不能盲目出价；
2. 线索可以帮助判断；
3. 不是每轮都要拍下，放弃也是策略。

### 21.2 第一局流程

- 第一轮：系统推荐安全价，让玩家成功小赚；
- 第二轮：出现赝品提示，让玩家学会避坑；
- 第三轮：引导玩家观察对手出价；
- 第四轮：允许玩家被轻微坑一次，但复盘讲清楚；
- 第五轮：给一次翻盘机会。

### 21.3 教学提示文案示例

```text
这类货柜可能有赝品，出价超过40,000后风险会明显提高。
```

```text
你不一定要赢下每次拍卖。让别人高价买走，也是一种胜利。
```

```text
本轮你亏损的主要原因是忽略了“釉色异常”这条线索。
```

## 22. 风险点与解决方案

### 22.1 风险：玩家觉得随机性太强

解决：

- 每轮至少给有效线索；
- 局后展示线索命中；
- 减少纯随机暴雷；
- 新手期降低赝品和破损惩罚。

### 22.2 风险：玩家不愿意竞价

解决：

- 设置行为任务，例如“参与2次竞价”；
- 低风险货柜提高收益；
- 明确放弃也有小补偿；
- 机器人带动节奏。

### 22.3 风险：玩家过度上头后流失

解决：

- 推荐停手价；
- 风险提示；
- 保险商角色；
- 单轮暴亏软限制；
- 局后复盘降低情绪不满。

### 22.4 风险：商业化被骂P2W

解决：

- 不卖局内金币；
- 不卖强力角色；
- 所有技能免费可得；
- 付费内容明确标注“仅外观”；
- 竞技模式统一初始资金。

### 22.5 风险：服务器结算争议

解决：

- 服务端权威；
- 交易流水；
- 对局回放；
- 客服查询码；
- 异常自动补偿。

## 25. 关键结论

这个项目的核心不是“做一个拍卖游戏”，而是做一个玩家能持续相信的多人心理博弈游戏。

真正的成败点有四个：

1. 局内公平，不能P2W；
2. 结算可信，不能吞币、黑箱、解释不清；
3. 亏损可复盘，让玩家觉得自己能进步；
4. 胜利强反馈，让捡漏、反坑、开稀有都有传播点。

最推荐的首版方向是：

```text
4人短局 + 三国珍宝包装 + 动物拟人角色 + 明拍/暗标/次高价拍卖 + 线索复盘 + 图鉴成长 + 外观商业化
```

不要急着做复杂交易市场、充值金币、强角色养成。先把“5轮一局，玩家觉得好笑、刺激、能复盘、愿意再来一局”做扎实。

- 闪拍制造短时间决策压力；
- 开出传说物形成传播点。

## 32. 样例局整体复盘

### 32.1 每轮功能定位

| 轮次 | 模式 | 功能 |
|---|---|---|
| 第1轮 | 明拍 | 建立规则理解和首次正反馈 |
| 第2轮 | 暗标 | 展示赝品、假情报和亏损复盘 |
| 第3轮 | 次高价 | 展示策略竞价和技能价值 |
| 第4轮 | 押金明拍 | 展示高风险、修复费和诱导成本 |
| 第5轮 | 闪拍 | 制造翻盘和高潮 |

### 32.2 情绪曲线

```text
第一轮：小赚，建立信心
第二轮：被坑，理解风险
第三轮：策略成功，看到深度
第四轮：高风险亏损，强化谨慎
第五轮：大反转，制造传播点
```

### 32.3 玩家学习点

| 学习点 | 对应轮次 |
|---|---|
| 线索能帮助判断 | 第1轮 |
| 公共高估值可能是陷阱 | 第2轮 |
| 假情报和玩家行为会误导 | 第2轮 |
| 技能不是直接答案，而是辅助判断 | 第3轮、第5轮 |
| 高价值物可能被修复费吃掉利润 | 第4轮 |
| 放弃也是策略 | 全局 |
| 落后仍有翻盘机会 | 第5轮 |

### 32.4 复盘界面需要突出的问题

局后复盘要让玩家看到：

- 自己哪一轮赚得最好；
- 哪一轮亏损最大；
- 亏损是因为出价过高、忽略风险、被假情报误导，还是判断线索错误；
- 是否有更优停手价；
- 哪个玩家成功诱导了别人；
- 是否有关键技能使用改变局势。

### 32.5 示例复盘文案

#### 对玩家C

```text
你在第二轮被假情报误导，亏损28,000；但在第三轮利用读心技能压过A，并在第五轮判断A保守后成功搏到高价值货柜，最终完成翻盘。
```

#### 对玩家A

```text
你前期判断稳定，第一轮成功捡漏，第二轮也避开赝品。但第五轮在已获得高价值线索的情况下出价偏保守，被C反超。
```

#### 对玩家B

```text
你第二轮和第四轮都成功制造误导，但没有在关键货柜中获得实际收益。后续可以在诱导他人同时保留自己的盈利机会。
```

#### 对玩家D

```text
你在第四轮使用保险时机合理，但出价仍超过净价值过多，导致保险没有有效止损。建议结合修复费线索降低最高出价。
```

## 34. 程序实现要点

### 34.1 核心服务端流程

```text
创建房间
→ 匹配玩家/机器人补位
→ 初始化随机种子
→ 生成本局货柜序列
→ 每轮生成藏品与线索
→ 下发公共信息和私人信息
→ 收集玩家技能与出价
→ 服务端判定成交
→ 服务端结算收益
→ 写入流水和回放
→ 下发结果
→ 最终排名和奖励结算
```

### 34.2 随机种子

每局生成一个 matchSeed。

所有关键随机都基于：

```text
matchSeed + roundIndex + boxID + playerID
```

用于保证：

- 可复盘；
- 可查证；
- 服务端和测试工具可重放；
- 防止客户端伪造。

### 34.3 客户端同步重点

客户端需要同步：

- 当前阶段；
- 阶段倒计时；
- 当前货柜公共信息；
- 自己私人线索；
- 玩家资金；
- 出价状态；
- 技能使用状态；
- 成交结果；
- 开箱表现；
- 结算结果。

### 34.4 阶段状态机

```text
WAITING
MATCHED
ROUND_PREPARE
BOX_SHOW
CLUE_PHASE
BID_PHASE
DEAL_PHASE
REVEAL_PHASE
ROUND_SETTLE
MATCH_SETTLE
FINISH
```

### 34.5 机器人决策简版

机器人出价可以用以下模型：

```text
估计价值 = 公共估值均值 * 公共可信度 + 私人线索修正 + 角色性格修正
最高可接受价 = 估计价值 * 风险系数
最终出价 = 最高可接受价 * 随机扰动
```

机器人性格参数：

| 参数 | 说明 |
|---|---|
| RiskPreference | 风险偏好 |
| BluffRate | 抬价/虚张概率 |
| ClueTrust | 对线索信任度 |
| StopLossRate | 止损倾向 |
| AggressiveLevel | 激进程度 |

## 35. 测试用例重点

### 35.1 功能测试

- 4人能正常进入房间；
- 机器人能补位；
- 每轮能正确生成货柜；
- 公共线索所有玩家一致；
- 私人线索只给对应玩家；
- 出价超资金时不能提交；
- 暗标不能提前看到别人出价；
- 次高价拍卖支付规则正确；
- 押金返还和扣除正确；
- 技能次数扣除正确；
- 断线重连后局面同步正确；
- 单轮结算正确；
- 最终排名正确。

### 35.2 数值测试

- 新手局平均亏损不能过高；
- 首局至少有一次明显正反馈；
- 赝品导致的大亏比例不能超过设定阈值；
- 各角色胜率不能长期偏离过大；
- 闪拍翻盘率不能过高；
- 次高价模式不能成为无脑最优；
- 保险商不能过强，也不能无存在感。

### 35.3 体验测试

- 玩家是否看懂公共估值；
- 玩家是否理解赝品；
- 玩家是否理解修复费；
- 玩家是否理解次高价规则；
- 玩家是否愿意看复盘；
- 玩家亏损后是否愿意再玩；
- 玩家是否觉得付费公平；
- 玩家是否能记住角色差异。
