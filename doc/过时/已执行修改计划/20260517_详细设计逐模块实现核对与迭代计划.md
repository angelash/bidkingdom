# 详细设计逐模块实现核对与迭代计划

> 日期：2026-05-17  
> 对照文档：`bitkingdom/doc/多人暗拍博弈游戏详细设计文档.md`  
> 对照代码：`bitkingdom/apps/*`、`bitkingdom/packages/*`  
> 结论：现有版本已经跑通“4人房间 + 服务端权威 + 5轮主流程 + 三种基础拍卖 + 线索/技能/机器人 + 开箱结算”的骨架，但内容厚度、样例局节奏、押金/闪拍、局外成长、复盘解释和新手引导明显不足。玩家体感为“主流程有了，游戏内容还没长出来”是准确的。

## 2. 当前代码已经做到的事

- `packages/config`：6个角色、100个藏品、30个货柜、5套套装、机器人性格配置、配置校验。
- `packages/match-core`：服务端权威对局状态、隐藏信息快照、公共/私人/技能/假情报线索、明拍/暗标/次高价、技能、机器人有限信息决策、开箱、结算、交易流水、事件日志。
- `apps/server`：Fastify + Socket.IO，创建/加入房间、机器人补位、开局、阶段定时、技能/出价/停手/表情、个人快照、事件/交易查询。
- `apps/web`：React可玩Demo，首页、房间、角色选择、局内看货、线索、出价、技能、开箱、结算、移动适配。

## 3. 最影响体验的缺口

### 3.1 没有“设计好的5轮情绪曲线”

文档第26-32章给了明确样例局：

```text
第1轮 明拍：稳定收益，建立信心
第2轮 暗标：赝品风险，假情报误导
第3轮 次高价：读心压价，展示策略
第4轮 押金明拍：修复费、诱导成本
第5轮 闪拍：传说物翻盘
```

当前代码随机抽货柜，虽然能玩，但无法保证第一局教学、情绪曲线和学习点。

### 3.2 复盘还不够“输得明白”

当前结算只显示成交、真值、利润。文档要求展示：

- 线索命中；
- 假情报来源；
- 亏损原因；
- 修复费/赝品/押金/保险拆解；
- 更优停手价；
- 谁诱导了别人。

### 3.3 拍卖模式少了押金和闪拍

文档首版完整样例实际用了5种节奏。当前只支持3种，导致第4、5轮的变化感不足。

### 3.4 角色系统缺少“每局次数”

当前只有冷却，和文档“每局可用2次/保险1次”不一致。

### 3.5 局外驱动基本没有

图鉴、任务、等级、最终局后评价都还没做。短期可以先做“对局内临时图鉴/任务反馈”，不必马上持久化。

## 4. 本轮迭代目标

优先补“玩起来像设计文档”的内容，而不是先做后台或商业化：

1. 增加文档样例5轮脚本局，默认Demo按样例顺序出货柜。
2. 增加 `押金明拍` 和 `闪拍` 两种模式。
3. 角色增加每局技能次数限制。
4. 结算数据增加押金、保险返还、复盘线索详情。
5. UI增加拍卖规则、出价记录、当前领先者、结算拆解、线索复盘、临时任务/图鉴反馈。
6. 增加押金/闪拍/脚本局测试。
