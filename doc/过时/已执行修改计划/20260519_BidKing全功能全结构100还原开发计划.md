本计划中的 100% 还原必须同时满足六条硬标准：

1. 配置表 100% 行为接入  
   52 张表不能只登记行数。每张表都必须有明确的运行层用途、服务端或前端 owner、字段解释、测试覆盖和验收证据。

2. 原类群 100% 映射  
   reverse 侧 `Scripts` 共有 1256 个 `.cs` 文件、约 201305 行。工程需要建立逐类映射账本：每个原类至少对应一个本项目模块、组件、服务、DTO、测试或“已审定无运行价值”的归档说明。不能存在未分类类。

3. 服务端 100% 权威状态  
   货币、票券、背包、道具、任务、成就、邮件、商店、排行榜、交易、协会、好友、活动、通行证、设置、敏感词过滤和比赛结算都必须由服务端 profile/ledger/event 驱动，前端不能自行编造权威状态。

4. 核心竞拍 100% 行为等价  
   `BidMap / Drop / Item / RankMap / RankAi / Hero / HeroSkin / Skill / SkillGroup / SkillEffect / BattleItem / Sim` 需要覆盖地图、掉落、轮次、成交线、机器人、角色技能、掌眼情报、战前道具、战中道具、结算和回放边界。

5. UI 链路 100% 可操作  
   `UIWnd / Guide / Sound / Language / DirtyWords / Emoji / Notice / ErrorCode` 不能只展示摘要，要进入窗口层级、入口开关、弹窗、引导、音频触发、本地化 key、输入过滤和错误提示。

6. 测试证据 100% 闭环  
   每个系统必须有单测、接口测试、socket 流程测试、Playwright UI 冒烟、配置 parity 校验、账本幂等测试和最终审查矩阵。没有测试证据的功能不算完成。

- 不接真实支付、真实线上交易、真实线上账号体系；支付/DLC/礼包只做本地模拟。

## 1. 总体执行策略

### 1.1 三条主线并行推进

1. 映射线

### 1.2 完成状态

所有计划项使用统一状态：

| 状态 | 含义 |
| --- | --- |
| `NotStarted` | 未实现或仅停留在展示 |

### 1.3 阻断规则

- 任一配置表没有 owner，不进入 100%。
- 任一原 `.cs` 类没有映射结论，不进入 100%。
- 任一经济变更不走 ledger，不进入 100%。
- 任一前端权威状态来自本地硬编码，不进入 100%。
- 任一入口只有卡片展示没有操作流，不进入 100%。
- 任一系统没有测试证据，不进入 100%。
