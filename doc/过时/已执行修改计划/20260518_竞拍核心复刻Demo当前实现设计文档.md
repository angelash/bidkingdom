# 竞拍核心复刻 Demo 当前实现设计文档

## 版本目标

本版本继续收敛到“珍宝局包装下的 BidKing 核心复刻”，不扩展商城、市场、成长和活动系统。当前实现目标是让内部 Demo 能真实跑通：

- 4 人同局，默认 1 名玩家 + 3 名 AI。
- 开局前固定选择 `明拍` 或 `暗拍`，整局不切换。
- 服务端先确定本局唯一藏宝仓，再下发候选仓列表播放随机滚动。
- 随机仓结果展示后，进入掌眼人情报揭示。
- 同一藏宝仓最多 5 轮渐进揭示、竞价、反馈。
- 角色信息技能在每轮开始自动补充私人视野，并能标记右侧格子。
- 最终成交后在右侧仓库格原地逐件开箱，收益动态变化。

## 启动与访问

- Web 固定端口：`http://127.0.0.1:5188`
- API 固定端口：`http://127.0.0.1:8787`
- 启动：`npm run dev:start`
- 停服：`npm run dev:stop`
- 重启：`npm run dev:restart`

启动脚本会清理旧的 `5188/8787/5177/8798` 端口占用，避免前端随机换端口。

## 房间流程

1. 首页输入掌柜名、房间码或 AI 数量。
2. 房主选择本局模式：
   - `暗拍`：普通对局只公开提交状态与排名。
   - `明拍`：公开出价记录与金额。
3. 创建房间后默认补齐 3 个 AI，共 4 人。
4. 房间内房主仍可切换明拍/暗拍；开始后锁定。
5. 点击开始后，服务端创建权威对局，进入随机仓阶段。

## 核心状态机

当前核心模式状态机为：

```text
lobby
→ warehouse_roll
→ warehouse_selected
→ auctioneer_reveal
→ intel
→ auction
→ settlement(本轮反馈，未成交)
→ 下一轮 intel / auction
→ reveal(最终原地开箱)
→ settlement(最终账务)
→ ended
```

实现位置：

- `packages/shared/src/index.ts`：`RoundPhase`、`openingCandidates`、`auctioneerClue`
- `packages/match-core/src/match.ts`：开局阶段、随机仓候选、掌眼情报、渐进揭示
- `apps/server/src/roomManager.ts`：阶段定时推进、AI 出价、提前结算

## 随机仓

核心模式不再把旧 `scriptedRounds` 拼成一个固定同仓，而是从 `config.containers` 中随机抽取真实仓型模板。本局唯一仓会保存在 `coreWarehouse`，之后 5 轮都复用同一个仓。

前端随机仓动画使用服务端下发的 `openingCandidates`：

- 候选卡从右向左滚动。
- 中央有 `SELECT` 选中框。
- 轨道使用减速曲线停在服务端选中的仓。
- 结果展示仓名、来源、标签和估值区间。

## 掌眼人情报

第一轮出价前会进入 `auctioneer_reveal` 阶段。服务端为本局抽一条公共掌眼情报，并随每轮快照下发：

- 随机抽验若干件藏品，点亮对应格子的轮廓、品质、品类。
- 或公布高品质候选数量，并标记部分高价值候选格。
- 或公布主要品类数量，并标记样本格。
- 或公布总占格数，并标记大件样本。

在 `warehouse_roll` 和 `warehouse_selected` 阶段，普通公共情报流暂不显示这条掌眼情报；进入 `auctioneer_reveal` 后，情报加入中央信息流，并影响右侧仓库格。

## 角色信息

每轮开始时，核心包会根据玩家角色生成自动私人线索：

- `鉴定师/估值类`：收窄本仓真实总值区间。
- `走私商/单品类`：标记高价值候选格，并显示品类和品质。
- `修复师/风险类`：标记可能有修复费或真伪问题的格子。
- `保险商/避坑类`：统计低价值或异常藏品数量与合计价值。
- `心理师/读局类`：根据上一轮领先者和公共估值给出心理价区间。

右侧仓库格由“公共渐进揭示 + 掌眼人公共标记 + 当前玩家私人技能标记”合成。不同玩家快照可能看到不同的标记。

## 单仓五轮成交

同一个藏宝仓最多推进 5 轮。每轮出价 60 秒，所有玩家提交或停手后立即结算本轮。

成交阈值：

| 轮次 | 成交条件 |
| --- | --- |
| 第 1 轮 | 最高价达到第二名 200%，即高出 100% |
| 第 2 轮 | 最高价达到第二名 160%，即高出 60% |
| 第 3 轮 | 最高价达到第二名 130%，即高出 30% |
| 第 4 轮 | 最高价达到第二名 110%，即高出 10% |
| 第 5 轮 | 最高价高于第二名即可成交 |

第 5 轮若最高价同价，则追加竞拍回合。加赛保持同仓、同模式、同一揭示层级，只解决同价。

## 明拍与暗拍

明拍：

- 顶部显示 `明拍`。
- 中间显示公开出价记录。
- 反馈区显示最高价、第二名金额、领先比例和成交线。

暗拍：

- 顶部显示 `暗拍`。
- 中间只显示未提交、已提交、结算后排名。
- 普通对局快照隐藏暗拍金额；后台复盘仍保留真实金额。

## 出价界面

底部动作栏只在 `auction` 阶段出现：

- `推荐价`：使用当前可见估值、线索和风险计算建议上限。
- `出价`：打开数字键盘。
- `停手`：本轮放弃。
- `技能`：手动技能入口，仍受冷却和次数限制。
- `情报`：打开实时情报弹窗。

出价必须经过数字键盘和二次确认后提交。

## 实时情报

当前实现了第一版静态候选藏品查询：

- 按品质筛选：杂项、普通、精品、稀有、传说、赝品。
- 按类别筛选：优先显示本轮已知类别，否则显示全配置类别。
- 按轮廓筛选：`1x1`、`2x1`、`2x2`。
- 列表显示图标、名称、类别、品质、轮廓和参考显示价。

该功能用于帮助玩家把“品质/品类/轮廓”线索转化为可能价值判断。

## 原地开箱

最终成交后进入 `reveal`：

- 右侧仓库格保持原布局，不另开区域展示藏品。
- 服务端逐件调用 `revealNextItem`。
- `revealDelayForItem` 按品质控制节奏：低品质快，高品质慢。
- 前端每个格子按稀有度播放不同长度的揭示动画。
- 收益面板显示最终竞拍价格、已开藏品价格和当前利润。
- 当前利润从 `-成交价` 开始，亏损红色，转正绿色。

## AI 行为

AI 只在 `auction` 阶段出价。估价依据包括：

- 公共估值区间。
- 自己的私人/技能价值线索。
- 公共风险线索。
- 风险偏好和溢价容忍。

明拍 AI 根据当前价 + 最小加价判断是否跟价；暗拍 AI 提交一次密封报价。全部玩家已提交或停手时，服务端立即结算。

## 自测状态

本轮已覆盖：

- 核心模式第一轮从 `warehouse_roll` 开始。
- 服务端下发随机仓候选。
- 掌眼情报在揭示阶段后进入公共情报流。
- 单仓 5 轮复用同一仓。
- 未达成交线进入反馈并推进下一轮。
- 第 5 轮成交和原地开箱仍可用。

```bash
npm run typecheck
npm run test
```

## 2026-05-18 表驱动兼容层补充

新增 `@bitkingdom/bidking-compat` 工作区包，按 BidKing 业务表名建立可提交的等价运行层：

- `BidMap`：125 行。
- `Item`：1136 行。
- `Drop`：629 行。
- `Hero`：20 行。
- `Skill`：256 行。
- `SkillGroup`：5 行。
- `SkillEffect`：36 行。
- `RankMap`：83 行。
- `RankAi`：120 行。
- `Map`：9 行。
- `BattleItem`：64 行。

当前核心模式已经从兼容 `BidMap + Drop + Item` 生成本局同仓、藏品掉落和右侧 Loot 格。成交线从兼容 `auction_rounds_rate` 读取，校验口径为 `[2000,1600,1300,1100,0]`。`RankMap` 已进入运行链路，用来控制核心拍卖轮次时长和最低出价。AI 出价参数已开始接入 `RankAi` 风格的风险、诈唬、溢价和进攻性参数。局内实时情报候选库也已切到兼容 `Item`，点击仓库格后按同一套品质、类别和轮廓候选反查。

`Hero + Skill + SkillEffect` 已进入核心玩法：

- 每名玩家按席位映射到兼容 `Hero`。
- 每轮自动情报从该 `Hero` 对应的 `Skill` 组生成，进入玩家私人视野。
- 手动技能也改为读取兼容 `Hero/Skill` 的技能组、轮次和 CD，并按 `SkillEffect` 效果键决定是估值、单品标记、读局、公共传闻、品质揭示还是风险信息。

拍卖师情报流程已从“直接抽一条”升级为“四张候选情报卡中抽中一张”：

- `auctioneerChoices` 下发 4 张候选卡。
- `auctioneerClue` 是最终抽中的掌眼人情报。
- 前端揭示 overlay 显示候选卡和选中卡，对应 BidKing 中随机披露情报的仪式感。

出价弹窗补齐了 BidKing 高频手感：

- 显示当前轮最低出价。
- 支持上一轮出价快捷填入。
- 支持隐藏/显示金额。
- 保留数字键、`00`、`000`、`x2.0`、归零和二次确认。

图鉴和收藏柜已改为兼容 `Item` 数据入口：

- 核心对局中开出的 `compat_*` 实例物品会归一到兼容图鉴 ID。
- 局外图鉴、收藏柜、详情页继续使用三国包装，但数据总量和品类入口来自兼容 `Item`。

兼容层只提交等价字段结构、数量校验和三国/古代包装后的运行数据，不提交原始商业表全文、反编译源码或资源本体。

## 当前边界

1. `BidMap + Drop + Item + RankMap + Hero + Skill + SkillEffect + RankAi` 已接入核心模式；后续只剩把更多 `SkillEffect` 细分成动画/音效表现，而不是核心规则缺口。
2. `RankAi` 参数已进入机器人出价调参，但还不是完整 AI 行为树。
3. 仓库格知识已支持公共/私人标记合成，但还不是完整 `knowledge.ts` 持久知识层。
4. 实时情报已可筛选兼容 `Item` 候选藏品，并已支持点击仓库格自动带入筛选；后续需要补更接近 BidKing 的多层情报页和来源分组。
5. 加赛轮已支持，但没有专属 UI 动画。
6. 安全、压测、账号、反作弊仍不在本轮目标内。
