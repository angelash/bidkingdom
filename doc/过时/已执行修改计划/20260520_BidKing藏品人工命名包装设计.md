# BidKing 藏品人工命名包装设计

## 问题诊断

当前藏品显示名直接来自兼容表的 `packaged_name`，原始格式基本是“品质 + 大类 + 格形 + ID”，例如“凡品府邸日用1格方件1011001”。这个名字只能说明数据结构，不能说明画面里是什么器物，也没有三国题材的识别点。

图片重复感的源头也在这里：源物品定义只提供大类、尺寸和 ID，缺少“香炉、鼎、胡床、青龙偃月刀、传国玉玺”这类强物象。只靠这些弱定义去生成图片时，很容易在同一大类里反复产出相近的瓶、鼎、盒、屏。

## 命名规则

1. 显示名统一为“品质感前缀 + 具体物象”，去掉格形、横竖件和数字 ID。
2. 低品质物品优先准确识别画面物件，例如铜香炉、青铜鼎、胡床、妆镜、药囊、铁矛头。
3. 高品质物品绑定三国人物、事件、地点或名件，例如吕布方天画戟、关羽青龙偃月刀、诸葛亮出师表、传国玉玺。
4. 同类重复图像用材质、纹样、用途拉开差异，例如“兽耳铜香炉”和“盘龙博山炉”不再都叫府邸日用。
5. 大类仍保留原表分类，用作筛选；名字负责让玩家一眼识别“这是什么”。

## 覆盖范围

本轮人工覆盖 `101-110` 藏品主线和 `1006001 / 1306001 / 1306002` 秘藏件：

- `101 府邸日用`：按当前图片轮廓逐个命名，覆盖香炉、鼎、胡床、宫灯、屏风、案几、坐榻等。
- `102 医药军需`：覆盖军医药箱、药瓶、针匣、麻沸散、青囊药匣等。
- `103 锦衣冠服`：覆盖发簪、步摇、香囊、纶巾、霞帔、人物衣冠。
- `104 军械兵装`：补齐青龙偃月刀、方天画戟、丈八蛇矛、双股剑、青釭剑等三国武库名件。
- `105 矿玉珠宝`：覆盖夜明珠、玉璧、玉玺、印材、金玺等国器线索。
- `106 文玩古器`：覆盖铜雀台瓦当、虎符、石经、博山炉、八阵碑等古器。
- `107 机巧器具`：覆盖木牛流马、诸葛连弩、司南、浑天仪、借风仪。
- `108 车马舟楫`：覆盖赤兔、的卢、绝影、楼船、车舆、马具。
- `109 酒食香料`：覆盖青梅煮酒、铜雀宴、军宴食盒、药膳酒壶。
- `110 书画典籍`：覆盖孟德新书、太平要术、出师表、隆中对、八阵图。

## 接入位置

人工表位于 `packages/bidking-compat/src/itemPackagingOverrides.ts`，由三个展示面统一读取：

- 藏品百科 `codexRuntime.ts`
- 实时情报 `LiveIntelPanels.tsx`
- 对局核心揭示 `match-core/src/bidking/compatRuntime.ts`

这样玩家在百科、候选情报、拍卖揭示里看到的是同一套命名，不会一个地方叫名件、另一个地方又退回机器占位名。

## 后续图片方向

后续如果继续处理重复图片，应把这张人工命名表作为图片提示词的核心输入，再补每件物品的材质、形制、时代纹样和可视角度。图片生成不应再只吃“大类 + 尺寸 + ID”，否则仍会回到同类器物反复撞脸的问题。
