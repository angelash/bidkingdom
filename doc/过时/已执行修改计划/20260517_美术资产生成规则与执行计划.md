# 珍宝局 BS Demo 美术资产生成规则与执行计划

> 日期：2026-05-17  
> 目标：把“AI 资产廉价感”和“资产不可追溯”提前压住，先建立可批量执行、可验收、可接入 Web Demo 的生图规则，再按清单生成与接入。

## 1. 总体方向

项目采用“**三国珍宝 + 动物拟人拍卖场**”包装：

- 场景：乱世商会、古玩拍卖厅、黑市密仓、旧宫急拍、军需旧库；
- 角色：动物拟人掌柜，轻策略桌游感，不能做成传统三国武将卡；
- 藏品：玉器、兵器、竹简、瓷器、珠宝、漆器、军需、赝品、破损件；
- 氛围：轻松互坑、识宝捡漏、开箱反转，避免厚重战争和现实货柜节目照搬。

## 2. 目录与版本规则

```text
apps/web/public/art/
  generated/                 # 原始生成图，只做存档，不直接在 UI 中引用
    backgrounds/
    roles/
    containers/
    items/
    ui/
  approved/                  # 人工/自动 QC 后可接入版本
    backgrounds/
    roles/
    containers/
    items/
    ui/
  manifest.json              # UI 实际引用的资产映射
```

命名格式：

```text
{category}_{semantic_key}_v{n}.{ext}
```

示例：

- `background_auction_hall_v1.png`
- `role_appraiser_portrait_v1.png`
- `container_sample_black_market_v1.png`
- `item_sample_r5_imperial_seal_v1.png`

生成图先进入 `generated`；通过验收后复制到 `approved`；UI 只读取 `manifest.json` 指向的 `approved` 资产。任何替换都新增版本，不覆盖旧图。

## 3. 全局提示词前缀

所有资产提示词必须包含以下约束：

```text
Use case: stylized-concept
Project style: light strategy board-game visual, Three Kingdoms treasure auction, anthropomorphic animal merchants, eastern black-market auction hall.
Visual language: clean silhouette, readable shapes, polished web-game asset, slightly stylized, not realistic gritty war art.
Palette: ink green, lacquer red, aged bronze gold, jade cyan, warm rice-paper highlights.
Lighting: warm top-left key light, soft ambient shadow, clear subject separation.
Constraints: no text, no watermark, no logo, no modern UI labels, no gore, no real historical celebrity likeness, no cheap neon over-saturation.
```

透明/切图资产优先要求主体居中、平面纯色背景；若需抠图，使用纯色背景后本地抠图，不能直接把不可控背景图接入 UI。

## 4. QC 标准

### 4.1 必过项

- 无文字、无水印、无随机符号；
- 风格符合“三国珍宝 + 动物拟人拍卖场”；
- 主体轮廓清晰，缩到 128px 仍可读；
- 光向统一，不能同屏资产明暗完全冲突；
- UI 引用图不得有明显 AI 手部崩坏、畸形角色或不可解释杂物；
- 背景图不能抢过 UI 信息层级，中心区域必须可压暗承载文字；
- 藏品图标必须主体居中，边缘干净，不能出现多件无关物。

### 4.2 评分

| 等级 | 分数 | 处理 |
|---|---:|---|
| A | 9.0+ | 进入 `approved`，可直接引用 |
| B | 8.0-8.9 | 进入 `approved`，后续可替换 |
| C | 7.0-7.9 | 只进 `generated`，需要二次生成或局部编辑 |
| D | < 7.0 | 进入 `rejected` 或不入库 |

## 6. 全量资产计划

机器可读执行清单已生成在：

```text
bitkingdom/doc/20260517_美术资产提示词执行清单.json
```

该清单由 `bitkingdom/tools/generate-art-prompt-manifest.ts` 从当前配置自动生成，当前包含 171 个待生成资产提示词：12 个角色资产、35 个货柜封面、121 个藏品图标、3 个背景/UI基础资产。

### 6.1 背景 3 张

- `background_auction_hall`：常规拍卖厅，首页和低/中风险局；
- `background_black_market`：黑市密仓，高风险与押金局；
- `background_palace_flash`：旧宫急拍，闪拍/传说局。

规格：`1920x1080` 或同等 16:9；中心可压暗；不出现文字。

### 6.2 角色 6 组

每个角色两类资产：

- 立绘：`768x1024`，3:4，角色选择/局外角色页；
- 头像：`512x512`，1:1，局内座位。

| 角色 | 内容 |
|---|---|
| 鉴定师 | 狐掌柜，放大镜，冷静精明，玉青/蓝色强调 |
| 走私商 | 鸦行商，羽帽/商队披风，神秘，紫色强调 |
| 心理师 | 鹿谋士，竹简/折扇，观察者，绿色强调 |
| 造谣者 | 狸说客，折扇/笑面，朱红强调 |
| 修复师 | 獾工匠，修复工具/铜锤，琥珀强调 |
| 保险商 | 龟账房，算盘/账簿，稳健，绿色强调 |

角色提示词模板：

```text
<全局提示词前缀>
Asset type: role portrait / role avatar
Primary request: anthropomorphic <animal> auction merchant, <role identity>, Three Kingdoms merchant clothing, <prop>, <personality expression>.
Composition: 3/4 view, upper body for portrait; head-and-shoulders for avatar; clean subject separation.
Color accent: <role color>.
Avoid: full battlefield armor, weapon brandishing, modern suit, text, watermark.
```

### 6.3 货柜/拍品封面 30 张

全量货柜封面按 `container.artKey` 生成；同来源可复用一个基础构图，再生成批次变体。首批优先 5 张样例局，其余按照 6 个来源模板扩展。

规格：`1024x768`，4:3；用于货柜展示阶段；构图必须给 UI 留出上方标题和底部估值的视觉空间。

### 6.4 藏品图标 121 张

按 `ItemConfig.iconKey` 一一生成，规格 `512x512`，主体居中。生成提示词由以下模板拼接：

```text
<全局提示词前缀>
Asset type: collectible item icon
Primary request: Three Kingdoms treasure auction item icon, <item.name>, category <item.category>, rarity <item.rarity>.
Details: show the object as a single centered collectible, clean readable silhouette, material details matching category.
State: if fake then subtly suspicious/artificial; if repairCost > 0 then visible crack or repair mark; if setId exists then premium set-piece finish.
Composition: centered object, 1:1, no text, no watermark, no extra unrelated objects.
```

### 6.5 UI 与特效 12 张

- UI 纹理 6：拍卖牌、结算牌、图鉴卡、任务牌、风险提示、回放牌；
- 开箱特效 6：普通、精良、稀有、传说、赝品、破损。

UI 纹理必须空中心、低对比、可叠文字；特效图必须避免遮挡藏品名和价格。

### Batch A：当前 Demo 直接接入

生成并接入：

### Batch B：角色页完整化

生成：

- 6 张角色立绘；
- 6 张角色头像；
- 24 个通用表情或 6 角色 × 4 表情。

### Batch C：藏品图鉴完整化

生成：

- 21 个样例局藏品图标；
- 100 个配置藏品图标；
- 每个套装 1 张套装收集完成徽章。

### Batch D：运营包装

生成：

- 战令/商城展示图；
- 收藏柜主题图；
- 排行榜装饰图。

## 8. 自动检查清单

每次生成后执行：

- 文件是否存在于 `generated`；
- 尺寸是否符合规格；
- 文件体积是否合理；
- 是否写入 `manifest.json`；
- UI 是否只引用 `approved`；
- 浏览器截图确认不遮挡文本；
- 最终提交必须包含资产路径、提示词和 QC 备注。
