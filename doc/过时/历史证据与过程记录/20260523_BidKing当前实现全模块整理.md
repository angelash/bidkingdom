## 当前实现的结构特征

- 当前实现是 Web 本地化 clean-room 复刻：React/Fastify/Socket.IO/TS 类型契约，与原 Unity/Protobuf/AssetBundle 架构不同。
- 原表数据已经大规模接入，但很多地方是“解释 helper + 本地模拟行为”，不是逐类逐方法同构。
- 对局核心在 `match-core`，服务端房间只是推进和广播状态；原源码则更多是客户端展示 + 服务器权威状态。
- 当前项目新增了后台审计、测试、Playwright、开发服务脚本，这些不是源码模块，但对复刻验证有价值。
- 最大技术债仍是实体库存：当前 profile 与 match-core 大量地方仍使用扁平 inventory/codex/holdings，未完全落成原 `StockContainerData -> StockBoxData -> ItemData` 生命周期。
