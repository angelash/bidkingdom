# 20260521 BidKing 逆向资料快速查询索引

> 用途：给本项目后续开发、审查和问题定位提供一个“先查哪里”的入口。本文只记录本地授权分析产物的路径、索引结构和查询方法，不复制反编译源码、原始图片、音频、模型或商业文本全文。

## 总体判断

当前逆向资料已经有基础索引，覆盖代码、配置表、字段、资源包、AssetBundle 对象、抽帧和当前 UI 对照。缺的是一个按问题组织的总入口；本文补齐这层查询路由。

核心链路：

```text
Steam 本地 BidKing 安装目录
  -> tools/extract_bidking_unity.py
  -> reverse/bidking/ 本地证据库
  -> doc/*.md 与 bidking_restore_*_matrix.md
  -> tools/generate-bidking-compat-data.mjs
  -> packages/bidking-compat + packages/match-core + apps/server + apps/web
```

## 快速查询命令

以下命令默认在仓库根目录执行。

查某个反编译类、方法或关键词先看索引：

```powershell
rg -n "Battle_Main|Auction|LoadTable|FirstTools" reverse/bidking/code/source_index.csv reverse/bidking/code/type_lists reverse/bidking/code/keyword_hits.txt
```

确实需要看本地授权反编译行为时，再窄范围查源码目录：

```powershell
rg -n "LoadTable|auction_rounds_rate|Base64ToTxt|DecryptRaw" reverse/bidking/code/decompiled
```

查一张表的行数、列数和哈希：

```powershell
rg -n "^BidMap\.txt,|^Item\.txt,|^UIWnd\.txt," reverse/bidking/config/tables_index.csv
```

查字段名和列号：

```powershell
rg -n "^BidMap,|^Item,|^Skill,|^UIWnd," reverse/bidking/config/table_schema_index.csv
```

查某个配置 ID：

```powershell
rg -n "^2101`t" reverse/bidking/config/tables_tsv/BidMap.txt
rg -n '"id": 2101|\"Id\": 2101' reverse/bidking/config/key_tables_json/BidMap.json
```

查可读核心表：

```powershell
rg -n "2101|未知快递|auction_rounds_rate" reverse/bidking/config/localized_core reverse/bidking/config/key_tables_json
```

查 UIWnd 到前端窗口的关系：

```powershell
rg -n "Battle_Main|BattlePrevPanel|PackagePanel|HeroPanel" reverse/bidking/config/tables_tsv/UIWnd.txt doc/bidking_restore_uiwnd_matrix.md apps/web/src/bidking/app/windowRegistry.ts
```

查资源包或 prefab 对象：

```powershell
rg -n "battle.data|battleintelligence|griditem|packagepanel|heropanel" reverse/bidking/resources/assetbundle_manifest_bundles.csv reverse/bidking/resources/assetbundle_object_index.csv reverse/bidking/resources/assetbundle_object_summary.json
```

查当前实现里是否已经接入某个表或字段：

```powershell
rg -n "auction_rounds_rate|BattleItem|SkillEffect|UIWnd|bidKingUIWndRuntime" packages apps
```

查测试证据：

```powershell
rg -n "review-snapshot|config parity|BattleItem|Condition|UIWnd|BidMap" apps packages
rg --files apps/web/playwright apps/server/tests packages/match-core packages/bidking-compat | rg "test|playwright"
```

## 维护规则

- 新增逆向抽取产物时，先更新 `reverse/bidking/extraction_summary.json` 或对应索引，再补本文路径。
- 新增配置表或字段解释时，同步更新 `packages/bidking-compat`、`doc/bidking_restore_table_matrix.md` 和后台 `review-snapshot`。
- 新增 UIWnd 映射时，同步更新 `windowRegistry`、`doc/bidking_restore_uiwnd_matrix.md` 和 Playwright 证据。
- 不把 `reverse/`、原始商业素材、反编译源码全文、音频、模型或视频放入提交。
