## 源码模块风险结论

- 源码的核心不是单一“拍卖算法”，而是服务端权威 `GameData`、Protobuf 协议、实体库存和配置表共同驱动。
- 最容易误判的地方是表级数据已经存在，但行为仍在服务端或 Unity UI 类中；只复制表不等于复制行为。
- 对局、仓库、收藏柜、交易、社交、公会、任务这些模块之间高度耦合，尤其是实体物品 UID 和 `StockContainerData` 会贯穿战斗、背包、送拍、回收和陈列。
- 资源与 UI 也是源码工程的一部分，但当前项目受分发边界限制，只能做 visual substitute 或自有资产替代，不能按原 AssetBundle 原样分发。
