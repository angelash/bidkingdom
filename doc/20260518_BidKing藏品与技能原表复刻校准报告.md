# BidKing 藏品与技能原表复刻校准报告

整理时间：2026-05-18

## 结论

- `Snipaste_2026-05-18_18-08-35.png` 的仓库详情只展示名称、银币价值、类型图标、藏品图、基础收益、占格预览、出售/来源和介绍文本，没有独立的“破损/赝品/修复费”玩法字段。
- 原 `Item` 表没有 `is_fake`、`repair_cost` 等机制字段；此前 Demo 中的破损、赝品、修复费是自创风险层，已从 BidKing 兼容藏品数据和核心展示中移除。
- 原 `slot_type` 是十位宽、个位高的占格编码，例如截图中的 `1106005` 为 `slot_type=32`，即 3x2 横向藏品；图片生成与仓库摆放必须按这个尺寸走。
- 截图中的 `《星空》` 对应原表 `Item.id=1106005`：`item_type_id=[110,106]`、`item_quality=6`、`base_value=170200`、`collection_coin=0.0215880263825469`，换算为 `77.7/小时`。
- 原角色技能链路是 `Hero.cast_type -> Skill.id -> skilltarget/skilltargetvalue -> SkillEffect.Category`，不是此前 Demo 的 5 组自创技能。

## 已落地

- `@bitkingdom/bidking-compat` 的 `Item/Hero/Skill/SkillEffect/SkillGroup` 已由本地 reverse 表机械生成：
  - `Item` 保留 1136 行原表 ID、类型数组、品质、价值、占格、收益、图标键、模型键等数值字段。
  - `Hero` 保留 20 行原表 ID、`cast_type` 技能数组、语音/BGM/访问字段，并只替换三国包装名。
  - `Skill` 保留 256 行原表目标、目标参数、效果位置、冷却、轮次等字段。
  - `SkillEffect` 保留 36 行 `EffectId/Category/Param`。
- BidKing 核心运行层已改为：
  - `displayValue = base_value`，与仓库银币价值一致。
  - `collection_coin * 3600` 展示为基础收益/小时。
  - `itemFootprint(slot_type)` 按十位/个位解析，支持原表最大占格，不再使用旧的 1-8 自创尺寸表。
  - 技能线索按原 `skilltarget` 和 `SkillEffect.Category` 解释为尺寸、数量、品质、类型、价格、均价、格均价等信息。
- 藏品百科/仓库详情已去掉赝品筛选和修复费展示，改为显示真值、展示估值、基础收益、占格和套装。

## 资产生成规则

全量三国包装藏品 Prompt 已生成到本地忽略目录：

`reverse/bidking/generated_prompts/bidking_sanguo_item_prompts.json`

本次清单覆盖 `item_type_id=100/101-110` 且 `slot_type>0` 的 590 件藏品。每条包含：

- 原表 `itemId`、类型、品质、`baseValue`、`collectionCoinPerHour`。
- 严格占格 `footprint`。
- 严格画布尺寸：单格基准 256px，画布为 `w*256 x h*256`。
- 输出路径建议：`apps/web/public/art/generated/bidking/items/bidking_item_<id>.png`。
- Prompt 要求透明 PNG、单物体、无 UI 框、无文字、无水印、无背景、不做破损/赝品状态。

## 后续注意

- 现在的提交只保存安全包装后的表结构和数值镜像，不提交原始商业本地化文本、原图、音频或模型。
- 大批量透明 PNG 需要按 Prompt 清单走图像生成流水线；当前代码已经为 `packaged_icon_key=bidking_item_<id>` 预留稳定键。
