## BidMap 行为字段

| 字段 | 设计意义 |
| --- | --- |
| `parent_map_id` | 所属大场景 |
| `bidder_number` | 本局要求人数，源码创建比赛时必须精确匹配 |
| `drop_group_id=[9999, groupId, min, max]` | 从 `Drop.group_id` 抽取藏品，抽取数量在 min/max 间 |
| `item_count_min/max` | 候选藏品数量范围，作为掉落数量兜底 |
| `map_time` | 每回合竞价秒数 |
| `auction_rounds_rate` | 成交线倍率，当前标准为 `[2000,1600,1300,1100,0]` |
| `map_random_skill` | 每回合公共地图技能组，指向 `SkillGroup` |
| `public_estimate_min/max_rate` | 内部估值上下限系数，当前对客户端默认隐藏 |
| `required_items` | 入场余额门槛，常见为银币门槛 |
| `currency_cost` / `item_cost` | 入场消耗，默认未知 BidMap 只看货币消耗 |
| `risk` / `packaged_tags` / `art_key` | 前端展示风险、标签和拍场图 |

## 成交线

源码使用 `getBidKingCloseThreshold(roundIndex)`，把 `auction_rounds_rate` 转成“最高价领先第二名的差距阈值”。

| 回合 | rate | 阈值 | 策划解释 |
| ---: | ---: | ---: | --- |
| R1 | 2000 | 100% | 最高价必须超过第二名 100% 才成交 |
| R2 | 1600 | 60% | 最高价必须超过第二名 60% 才成交 |
| R3 | 1300 | 30% | 最高价必须超过第二名 30% 才成交 |
| R4 | 1100 | 10% | 最高价必须超过第二名 10% 才成交 |
| R5 | 0 | 0% | 有最高有效价且不平价即可成交；若最终轮平价会进入追加回合 |

追加回合限制：`round.index > 4` 时，玩家新出价必须高于自己上一轮出价。
