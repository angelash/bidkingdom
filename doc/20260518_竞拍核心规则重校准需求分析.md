# 竞拍核心规则重校准需求分析

## 依据

本轮以用户补充截图和 `doc/videos` 录屏观感为准，纠正当前 Demo 中“出价区错乱、明暗拍混用、房间进入流程缺少随机仓、轮间排名表达不足”的问题。

## 核心规则

1. 对局开始前由房主决定本局是明拍还是暗拍，5 轮内不切换模式。
2. 每轮出价阶段倒计时 60 秒。
3. 所有玩家都提交出价后，立即结束本轮出价阶段。
4. 每轮结算出价排名，排名写入左侧玩家列表，并保留 5 轮排名轨迹。
5. 明拍显示出价记录和当前领先价；暗拍不显示具体金额，只显示提交状态与轮次排名。
6. 成交阈值：
   - 第 1 轮：最高价高于第二名 100% 才直接成交。
   - 第 2 轮：最高价高于第二名 60% 才直接成交。
   - 第 3 轮：最高价高于第二名 30% 才直接成交。
   - 第 4 轮：最高价高于第二名 10% 才直接成交。
   - 第 5 轮：最高价高于第二名即可成交。
7. 第 5 轮若最高价与第二名相同，进入额外竞拍回合。Demo 当前先实现为自动开启加赛轮，保持同仓、同模式、继续 60 秒出价。

## 房间与开局

1. 创建房间前和房间内都提供明拍/暗拍模式选择；仅房主可在房间内改模式。
2. 开始对局后，进入地图/仓库随机抽选表现。
3. 抽到的仓库决定背景图、仓库名称、战利品轮廓。
4. 拍卖师技能/信息在开局后立即揭示一条情报，显示在中央情报列表。
5. 玩家可以在同一主界面中查看情报、使用技能、提交出价。

## UI 重点

1. 左侧玩家列表显示：
   - 玩家头像/角色/资金。
   - 当前轮是否已经出价。
   - 每轮排名 1-5 的轨迹；已结算显示名次，未结算置灰。
2. 中间主区域显示：
   - 当前轮、明拍/暗拍、倒计时。
   - 拍卖师/技能/仓库情报。
   - 明拍出价记录或暗拍提交状态。
   - 本轮排名反馈。
3. 右侧显示战利品仓库格。
4. 出价区改为弹窗流程：
   - 底部只保留主要动作按钮，避免挤压错乱。
   - 点击出价后打开数字键盘。
   - 点击确认出价后再弹出二次确认。

## 边界

1. 明拍最低加价仍使用 `minIncrement`，避免重复提交无效低价。
2. 暗拍允许玩家修改自己的暗标，最后一次提交为准。
3. 未提交的玩家在倒计时结束时按 0 处理。
4. 所有人提交后立即结算，但明拍不再因为只剩一个活跃玩家就提前结算。
5. 加赛轮不增加仓库揭示层级，只解决第 5 轮同价无法成交的问题。
